## 1. WHAT IS MISSING DI?

**Missing DI** (Missing Data Interpretation) is a special format of DI questions where the chart, table, or graph is **intentionally incomplete** — one or more data values are replaced by a **blank (?)** or **marked as unknown**.

- The missing value is **not directly readable** from the chart
- You must **use given clues** (row totals, column totals, averages, ratios, percentages, growth rates) to **compute the missing value first**
- Once the missing value is found, the actual DI questions (ratio, %, average, difference) can be answered

```
Example — Table with Missing Value:
| Year | Company A | Company B | Company C | Row Total |
|------|-----------|-----------|-----------|-----------|
| 2019 |    240    |     ?     |    180    |    600    |
| 2020 |    280    |    200    |    210    |    690    |

The "?" must be found before answering any question about Company B in 2019.
Row total = 600. Company B 2019 = 600 − 240 − 180 = 180.
```

---

### Where It Appears

```
Missing DI appears in:
→ IBPS PO / SBI PO (both Prelims and Mains) — 2–3 Missing DI sets per paper
→ IBPS Clerk / SBI Clerk — 1–2 Missing DI sets
→ RBI Grade B — Advanced Missing DI
→ SSC CGL Tier II — Missing table DI
→ UPSC CSAT — Paragraph with missing values
→ CAT — Data sufficiency + missing data combined

In banking exams, Missing DI typically carries 10–15 marks per paper
and is considered MODERATE to HARD difficulty.
```

---

### What Makes It Special

```
In normal DI: Read value → Apply formula → Answer.
In Missing DI: Find missing value → Read value → Apply formula → Answer.

The extra step (finding the missing value) is the KEY SKILL.
If you find the missing value wrong → ALL subsequent answers are wrong.

This means: precision in Step 1 (finding missing value) is absolutely critical.
```

---

## 2. WHY MISSING DI IS UNIQUELY CHALLENGING

```
CHALLENGE 1 — Hidden Extra Step:
Students forget to find the missing value first.
They try to answer questions directly and get stuck or guess wrong.
Prevention: ALWAYS find missing values BEFORE reading any question.

CHALLENGE 2 — Multiple Missing Values:
Some DI sets have 2 or 3 missing values.
You must find them in the RIGHT ORDER (some depend on others).
Prevention: Find independent missing values first, then dependent ones.

CHALLENGE 3 — Disguised Clues:
The clue to find the missing value is sometimes given:
→ In the question itself ("If total is 1200...")
→ As a note below the chart ("Row totals are as given")
→ As a percentage mentioned in passing
Students miss these clues and think the missing value is truly missing.

CHALLENGE 4 — Chain Errors:
If you find the missing value wrong, EVERY subsequent question using it is wrong.
This can cost you 4–5 marks from one arithmetic error.
Prevention: After finding missing value, cross-verify using a second method if possible.

CHALLENGE 5 — Visual Distraction:
In bar graphs and pie charts, the missing bar/sector is simply absent.
You might not even notice it's missing until you can't answer a question.
Prevention: Before reading questions, scan the chart for blanks or absent elements.
```

## 3. TYPES OF MISSING DI QUESTIONS

---

### Type 1 — Single Missing Value (Easiest)

```
One cell/bar/sector is missing. One clue is given. Direct computation.

Example:
Bar graph shows 5 years of data. 2020's bar is missing.
Clue: Total production over all 5 years = 1200 units.
Given: 2017=200, 2018=240, 2019=220, 2021=260.
Missing: 2020 = 1200 − (200+240+220+260) = 1200 − 920 = 280.
```

---

### Type 2 — Missing Value Embedded in Questions

```
The missing value is not found upfront — it's revealed within a specific question.

Example:
"If total students in 2020 = 500 and % in Science = 40%, find Science students."
Science = 40% × 500 = 200. The "?" in the chart is answered by this specific question.
The next question may give a different clue for a different missing value.
```

---

### Type 3 — Two Missing Values (Intermediate)

```
Two values are missing. Two clues are given.
Must find both (sometimes one is needed to find the other).

Example:
Row total = 800. Column total for Company B = 600.
Two unknowns (B's values in two years). Two equations → solve.
```

---

### Type 4 — Missing Value from Percentage Clue

```
A percentage relationship reveals the missing value.

Example:
Company A's 2020 revenue = 120% of Company A's 2019 revenue.
2019 revenue is given = 250. 2020 revenue (missing) = 250 × 1.20 = 300.
```

---

### Type 5 — Missing Value from Ratio Clue

```
A ratio relationship between known and unknown values.

Example:
The ratio of Company B's 2019 to 2020 revenue = 3:4.
2019 = 180 (given). 2020 = 180 × (4/3) = 240 (missing).
```

---

### Type 6 — Missing Value in Pie Chart (Sector Missing)

```
One sector's percentage or degree is missing.
Clue: All sectors must sum to 100% (or 360°).
Missing sector = 100% − (Sum of all given sectors).

Example:
Sectors: A=30%, B=25%, C=20%, D=?%, E=10%
D = 100 − 30 − 25 − 20 − 10 = 15%.
```

---

### Type 7 — Missing Value Requires Two-Step Reasoning

```
The missing value cannot be found in one step.
Need to first derive an intermediate value, then find the missing value.

Example:
"Company X had 25% profit margin in 2020. Its profit = ₹300 crores."
Revenue = Profit / Margin = 300 / 0.25 = ₹1200 crores. (Missing revenue found in 2 steps.)
```

## 5. CONCEPT 1 — THE MASTER FRAMEWORK FOR MISSING DI

---

### 5.1 The FIND Method

**Apply this four-step method to every Missing DI set, in order, before answering any question.**

```
F — Find: Locate ALL blanks / missing values in the chart. Count them.
I — Identify: For each missing value, identify the clue that reveals it.
N — Navigate: Find values in the correct order (independent first).
D — Derive: Compute each missing value. Cross-verify if possible.

Only AFTER completing FIND → read and answer the questions.

TIME INVESTED: 1–3 minutes on FIND.
BENEFIT: Every question answered correctly in under 30 seconds.
```

---

#### STEP F — Find All Blanks

```
Before reading a single question:
→ Scan the ENTIRE chart (every row, every column, every bar, every sector).
→ Mark every cell/position that has "?" or is absent or is labelled "unknown."
→ Count: How many missing values are there?

For a table:
→ Read row by row, left to right.
→ Any cell with "?" = missing value.
→ Note: Which row and which column is it in?

For a bar graph:
→ Are all bars present? Is any bar height unlabelled?
→ Compare number of categories to number of bars — missing if fewer bars than categories.

For a pie chart:
→ Do all sectors sum to 100%? If not, one sector's % is missing.
→ Is any sector unlabelled or shown without a percentage?

For a line graph:
→ Are all data points marked? Any gap in the line?
→ Is any point's value not labelled?

CRITICAL HABIT: Never start answering questions without first doing Step F.
```

---

#### STEP I — Identify the Clue for Each Missing Value

```
For each missing value found in Step F:
Ask: "What clue in this DI set tells me this missing value?"

CLUE SOURCES:
1. Given totals in the chart (Total row at bottom, Total column on right).
2. Notes below the chart ("All totals are as shown in the rightmost column").
3. The question stem itself ("If Company A's total = 1500, find...").
4. Percentage relationships ("A is 20% of the total row").
5. Ratio clues ("B's 2020 value is 4/3 of its 2019 value").
6. Growth rate clues ("Production grew by 15% from 2019 to 2020").
7. Derived relationships ("Profit = Revenue − Cost; profit is given").
8. Sector sum = 100% (for pie charts).
9. Average given ("Average of column B over all years = 250").

CRITICAL: If you cannot identify a clue for a missing value, it may be:
→ Given in one of the questions (scan all question stems).
→ Derivable from two other clues combined.
→ Truly not determinable (then it only needs to be found for specific questions).
```

---

#### STEP N — Navigate the Order

```
When there are MULTIPLE missing values:

INDEPENDENT missing values: Can be found directly from the clue, without needing another missing value.
DEPENDENT missing values: Require another missing value to be found first.

RULE: Find INDEPENDENT missing values FIRST.
      Then use their computed values to find DEPENDENT missing values.

Example:
Missing value 1 (MV1): In row A. Clue: Row A total = 800. All other values in row A given.
Missing value 2 (MV2): Column B total = 1200. MV1 is in column B.

MV1 is independent (only needs row A values + row total).
MV2 depends on MV1 (column B total includes MV1's cell).

Order: Find MV1 first → Then find MV2 using MV1.

If you try MV2 before MV1, you'll be stuck because the column total includes the unknown MV1.
```

---

#### STEP D — Derive and Cross-Verify

```
DERIVE: Use the identified clue to compute the missing value.
→ Apply the formula: Missing = Total − Sum of known values.
→ Or: Missing = Given% × Total.
→ Or: Missing = Known value × Growth rate.
→ Or: Solve the algebraic equation.

CROSS-VERIFY (when possible):
After finding a missing value, verify it using a SECOND clue if available.

Example:
Missing value in row 2019, column B.
Clue 1: Row 2019 total = 600. Other row values: 240, 180. B = 600−240−180 = 180.
Clue 2: Column B total = 540. Other column B values: 2018=150, 2020=210. 150+180+210=540. ✓

If Clue 2 also gives 180 → VERIFIED. Proceed confidently.
If Clue 2 gives a different value → ERROR somewhere. Recheck.

Cross-verification is the most powerful tool in Missing DI.
Always use it when a second clue is available.
```

---

### 5.2 Types of Clues Used to Find Missing Values

```
CLUE TYPE 1 — Row/Column Total (Most Common)
Formula: Missing = Total − Sum of all known values in that row/column.

Example:
Row total = 900. Known values: 200, 150, 250. Missing = 900−200−150−250 = 300.

CLUE TYPE 2 — Grand Total
Formula: Missing = Grand Total − Sum of all known cells.
(When only one cell is missing from the entire table.)

CLUE TYPE 3 — Average
Formula: Total = Average × Count.
Missing = (Average × Count) − Sum of known values.

Example:
Average of 5 values = 60. Sum = 60×5 = 300.
Known 4 values sum = 250. Missing = 300−250 = 50.

CLUE TYPE 4 — Percentage Share
Formula: Missing = (Given %) × (Total).

Example:
Missing value = 20% of row total (700). Missing = 0.20×700 = 140.

Or reverse: If missing's % is not given directly:
Missing % = 100% − Sum of all known percentages. Then: Missing = Missing% × Total.

CLUE TYPE 5 — Ratio Between Values
Formula: Missing = Known value × (Ratio numerator / Ratio denominator).

Example:
Missing:Known = 4:3. Known = 180. Missing = 180 × (4/3) = 240.

CLUE TYPE 6 — Percentage Change from Known Value
Formula: Missing = Known value × (1 + %change/100).

Example:
Missing = 2020 value. 2019 value = 250. Growth = 20%.
Missing = 250 × 1.20 = 300.

CLUE TYPE 7 — Derived Relationship
Formula: Derived = Component1 ± Component2.
Missing component = Derived ± Other component.

Example:
Profit = Revenue − Cost. Profit = 80 (given). Revenue = 300 (given).
Cost (missing) = Revenue − Profit = 300−80 = 220.

CLUE TYPE 8 — Sector Sum (Pie Charts)
Formula: Missing sector % = 100% − Sum of all given sector percentages.
Or: Missing sector degrees = 360° − Sum of all given sector degrees.

CLUE TYPE 9 — Question Stem Clue
Formula: The question itself tells you the total, ratio, or % needed.
Apply the clue given in the question to find the missing value.
```

---

### 5.3 The Missing Value is Never Truly Missing

```
KEY INSIGHT FOR EXAM CONFIDENCE:
In every properly designed Missing DI question, the missing value CAN be found.
The clue is ALWAYS there — either in the chart, in a note, or in the question.

Your job is to FIND the clue, not to guess or estimate.

If you cannot find the missing value:
1. Re-read ALL parts of the chart (every row header, column header, note).
2. Re-read ALL question stems (the clue may be in Q3 even if Q1 asks about it).
3. Check if there is a second missing value that must be found first.
4. Confirm: is this value actually missing, or is it just hard to read?

"Missing" in Missing DI = hidden, not absent.
```

## 6. CONCEPT 2 — MISSING VALUES IN TABLES

> Tables are the most common format for Missing DI in banking exams. Master these completely.

---

### 6.1 Row Total Clue

```
SCENARIO: One cell in a row is missing. The row total is given.

FORMULA: Missing cell = Row total − Sum of all other known cells in that row.

VISUAL:
| Year | A  | B  | C  | ?  | Total |
|------|----|----|----|----|-------|
| 2019 | 200| 150| 180| ?  |  620  |

Missing = 620 − 200 − 150 − 180 = 620 − 530 = 90.

STEP-BY-STEP:
Step 1: Identify the row with the missing cell.
Step 2: Identify which column has the row total.
Step 3: Sum all known cells in that row.
Step 4: Missing = Row Total − Known Sum.
Step 5 (verify): Does the computed value make sense? (Positive? Within reasonable range?)
```

---

### 6.2 Column Total Clue

```
SCENARIO: One cell in a column is missing. The column total is given.

FORMULA: Missing cell = Column total − Sum of all other known cells in that column.

VISUAL:
| Year | Column B |
|------|----------|
| 2017 |    120   |
| 2018 |     ?    |
| 2019 |    150   |
| 2020 |    180   |
| Total|    540   |

Missing (2018, B) = 540 − 120 − 150 − 180 = 540 − 450 = 90.
```

---

### 6.3 Average Clue

```
SCENARIO: The average of a row or column is given. One value is missing.

FORMULA:
Step 1: Total = Average × Count (number of values in the row/column).
Step 2: Missing = Total − Sum of known values.

VISUAL:
Values: 40, ?, 60, 80, 70. Average = 60.
Total = 60 × 5 = 300.
Missing = 300 − (40+60+80+70) = 300 − 250 = 50.

SHORTCUT:
If all other deviations from the average are known:
Missing deviation = −(Sum of all other deviations).
Missing value = Average + Missing deviation.

Example: Avg = 60. Others: 40(dev=−20), 60(dev=0), 80(dev=+20), 70(dev=+10).
Sum of other deviations = −20+0+20+10 = +10.
Missing deviation = −10. Missing value = 60−10 = 50. ✓
```

---

### 6.4 Percentage / Ratio Clue

```
SCENARIO: The missing value's percentage of a total, OR its ratio to another value, is given.

FORMULA (% clue):
Missing = (Given %) × Total.

Example:
Missing value = 15% of row total (800). Missing = 0.15 × 800 = 120.

FORMULA (ratio clue):
If Missing : Known = m : n → Missing = (m/n) × Known.

Example:
Missing : 2019 value = 5 : 4. 2019 value = 200.
Missing = (5/4) × 200 = 250.

FORMULA (% change clue):
If missing is X% MORE than known:
Missing = Known × (1 + X/100).

If missing is X% LESS than known:
Missing = Known × (1 − X/100).
```

---

### 6.5 Multi-Cell Missing (Two or More Missing Values)

```
SCENARIO: Two or more cells are missing. Two or more clues are given.

STRATEGY:
1. For each missing cell, identify its clue.
2. Determine which missing cells are INDEPENDENT (can be found without other missing cells).
3. Find independent ones first.
4. Then use those values to find dependent ones.

EXAMPLE — Two Missing Values (Both Independent):
| Year | A  | B  | C  | Row Total |
|------|----|----|----|----|
| 2018 | 200|  ? | 180|   600     |
| 2019 | 250| 210|  ? |   690     |

MV1 (2018, B): Row total = 600. B_2018 = 600−200−180 = 220.
MV2 (2019, C): Row total = 690. C_2019 = 690−250−210 = 230.
Both independent. Find in any order.

EXAMPLE — Two Missing Values (One Dependent):
| Year | A  | B  | C  | Row Total | Col B Total |
|      |    |    |    |           |     600     |
| 2018 | 200|  ? | 180|   600     |             |
| 2019 | 250|  ? | 230|   720     |             |

MV1 (2018, B): Row 2018 total = 600. B_2018 = 600−200−180 = 220.
MV2 (2019, B): Col B total = 600. B_2019 = 600−220 = 380.

MV2 DEPENDS on MV1. Must find MV1 first.
```

## 7. CONCEPT 3 — MISSING VALUES IN BAR GRAPHS

---

### Structure of Missing Bar DI

```
A bar graph has N categories (years/months/companies).
One (or more) bars are missing — their height is not drawn or labelled with "?".
A clue is given to find the missing bar's value.

COMMON CLUES IN BAR GRAPH MISSING DI:
→ "Total production over all years = X units."
→ "Average revenue per year = ₹Y crores."
→ "The missing year's production is 20% more than the previous year."
→ "The ratio of missing year to [known year] = 5:4."
→ "Sum of first 3 years = Z."
```

---

### Complete Example — Missing Bar Graph

```
BAR GRAPH: Annual Revenue (₹ Crores)
2017: 300 | 2018: 400 | 2019: ? | 2020: 350 | 2021: 500

Clue: Total revenue over all 5 years = 1700 crores.

FIND:
Step 1: Sum of known values = 300+400+350+500 = 1550.
Step 2: Missing (2019) = 1700 − 1550 = 150 crores.

VERIFY: Does 150 make sense? A significant drop from 400. Possible if context mentions a recession/disruption. ✓

ANSWERS:
Q: "What is the ratio of 2018 to 2019 revenue?"
A: 400:150 = 8:3. (HCF=50)

Q: "By what % did revenue change from 2019 to 2020?"
A: [(350−150)/150] × 100 = (200/150) × 100 = 133.33% increase.

Q: "What is the average annual revenue?"
A: 1700/5 = 340 crores.
```

---

### Multiple Missing Bars

```
TWO MISSING BARS WITH TWO CLUES:

BAR GRAPH: Production (lakh units)
2017: 120 | 2018: ? | 2019: 160 | 2020: ? | 2021: 200

Clue 1: Average production = 160 lakh units.
Clue 2: 2020 production = 125% of 2019 production.

FIND MV2 (2020) first — it's independent:
MV2 (2020) = 125% × 160 = 1.25 × 160 = 200.

FIND MV1 (2018) using Clue 1:
Total = 160 × 5 = 800.
MV1 (2018) = 800 − (120+160+200+200) = 800 − 680 = 120.

CROSS-VERIFY: Average = (120+120+160+200+200)/5 = 800/5 = 160. ✓
```

## 8. CONCEPT 4 — MISSING VALUES IN PIE CHARTS

---

### Structure of Missing Pie DI

```
A pie chart has N sectors. One (or more) sectors are missing their % or degree value.
The clue: All sectors must sum to 100% (or 360°).
Missing sector = 100% − Sum of all given sector percentages.
OR: Missing sector degrees = 360° − Sum of all given degrees.
```

---

### Single Missing Sector

```
EXAMPLE:
Pie Chart: Distribution of Budget
Education: 30% | Health: 20% | Defence: 25% | Transport: ? | Others: 10%

FIND:
Missing (Transport) = 100% − (30+20+25+10) = 100% − 85% = 15%.

THEN ANSWER QUESTIONS:
"If total budget = ₹8000 Cr, what is Transport allocation?"
Transport = 15% × 8000 = ₹1200 Cr.

"What is the ratio of Education to Transport?"
30% : 15% = 2:1.
```

---

### Missing Sector with Given Total

```
EXAMPLE:
Pie Chart: Employees by Department (Total = 2400)
Sales: 30% | IT: 25% | HR: ? | Finance: 20% | Ops: 10%

FIND:
HR % = 100 − (30+25+20+10) = 15%.
HR employees = 15% × 2400 = 360.

ANSWER QUESTIONS:
Q: "How many IT employees?"
IT = 25% × 2400 = 600.

Q: "IT employees are what % more than HR employees?"
(Same total → use % directly): [(25−15)/15] × 100 = 66.67%.
```

---

### Double Pie Chart with One Missing Sector Each

```
PIE CHART 1 (Year 2018, Total = ₹5000 Cr):
A: 30% | B: 25% | C: ? | D: 20% | E: 10%
C_2018 % = 100−(30+25+20+10) = 15%.
C_2018 value = 15% × 5000 = ₹750 Cr.

PIE CHART 2 (Year 2022, Total = ₹8000 Cr):
A: 25% | B: ? | C: 20% | D: 18% | E: 12%
B_2022 % = 100−(25+20+18+12) = 25%.
B_2022 value = 25% × 8000 = ₹2000 Cr.

THEN COMPARE:
"By what % did C's allocation change from 2018 to 2022?"
C_2022 value = 20% × 8000 = 1600. C_2018 = 750.
% change = [(1600−750)/750] × 100 = 113.33%.
```

## 9. CONCEPT 5 — MISSING VALUES IN LINE GRAPHS

---

### Structure of Missing Line DI

```
A line graph has data points for each time period.
One (or more) data points are missing — not labelled or shown as "?".
The clue is needed to compute the missing point's value.

COMMON CLUES:
→ "Total over all years = X."
→ "Average = Y."
→ "The missing year's value is 15% more than the previous year."
→ "The missing year's growth rate is the average of the adjacent years' growth rates."
→ "The ratio of missing to known = m:n."
```

---

### Complete Example — Missing Line Graph

```
LINE GRAPH: Company Revenue (₹ Crores)
2017: 200 | 2018: 250 | 2019: ? | 2020: 300 | 2021: 350

Clue: The revenue in 2019 was the average of 2018 and 2020 revenues.

FIND:
2019 = (250 + 300) / 2 = 275.

CROSS-VERIFY: Does the line look reasonable? 200 → 250 → 275 → 300 → 350. Smooth rise. ✓

ANSWER QUESTIONS:
Q: "By what % did revenue increase from 2018 to 2019?"
[(275−250)/250] × 100 = (25/250) × 100 = 10%.

Q: "Total revenue over 5 years = ?"
200+250+275+300+350 = 1375 crores.

Q: "What is the average annual revenue?"
1375/5 = 275 crores.
```

---

### Missing Line + Growth Rate Clue

```
LINE GRAPH: Production (thousand units)
2015: 100 | 2016: 120 | 2017: ? | 2018: 168 | 2019: 210

Clue: Production grew at the same rate from 2015 to 2016 as from 2016 to 2017.

FIND:
Growth rate 2015→2016: [(120−100)/100] × 100 = 20%.
Apply same rate 2016→2017: 2017 = 120 × 1.20 = 144.

VERIFY: Does 168 fit after 144? Growth 2017→2018: [(168−144)/144]×100 = 16.67%.
Is this consistent with the general trend? Yes (growth rate decreasing slightly). ✓

ANSWER QUESTIONS:
Q: "By what % did production increase from 2017 to 2018?"
[(168−144)/144] × 100 = (24/144) × 100 = 16.67%.
```

## 10. CONCEPT 6 — MISSING VALUES IN COMBINATION / MIXED CHARTS

---

### Table + Bar Graph (Most Common Mixed Missing DI)

```
SETUP:
Bar graph: Total values for each year (overall totals).
Table: % distribution by category (e.g., by department or product).
One bar's value OR one table % is missing.

HOW TO FIND THE MISSING VALUE:

If BAR value is missing:
→ Use table: compute the total from known category values.
→ Total = Sum of all category values for that year.
OR
→ Use the "total of all years" clue: Missing bar = Grand total − Sum of known bars.

If TABLE % is missing:
→ All row %s must sum to 100%.
→ Missing % = 100% − Sum of known %s in that row.

EXAMPLE:
Bar graph: Total students per year.
2018: 2000 | 2019: ? | 2020: 2500.

Table: % distribution by stream.
| Year | Science | Commerce | Arts |
|------|---------|----------|------|
| 2018 |   40%   |   35%    | 25%  |
| 2019 |   45%   |    ?     | 20%  |
| 2020 |   38%   |   32%    | 30%  |

Clue 1: Commerce 2019 students = 700.
Clue 2: Total students over all 3 years = 6900.

Finding missing table % (Commerce 2019):
Commerce 2019 = 700. Total 2019 = ?
Commerce 2019 % = Commerce students / Total 2019.
Need Total 2019 first.

Finding missing bar (2019):
Grand total = 6900. 2018+2020 = 2000+2500 = 4500. Total 2019 = 6900−4500 = 2400.

Now: Commerce 2019 % = 700/2400 × 100 = 29.17%.
Check: 45%+29.17%+20% = 94.17% ≠ 100%. Inconsistency → redo.

Correct approach: Missing table % = 100−45−20 = 35%.
Commerce 2019 students = 35% × 2400 = 840 (not 700 as given). 
This reveals that the given clue "Commerce 2019 = 700" was specific to a question, 
not a general chart clue. The chart's missing % is found by 100−45−20=35%.
```

## 11. CONCEPT 7 — MISSING DI WITH CASELET CLUES

---

### When the Clue Is in the Paragraph

```
Some Missing DI sets provide additional data in a CASELET (paragraph).
The caselet reveals the missing value(s) through text.

STRATEGY:
Step 1: Read the chart — identify all missing values.
Step 2: Read the caselet — extract the relevant clue for each missing value.
Step 3: Compute missing values using caselet clues.
Step 4: Answer questions from the complete chart.

EXAMPLE:
Table: Revenue of Companies A, B, C (₹ Crores)

| Year | A   | B   | C   | Total |
|------|-----|-----|-----|-------|
| 2019 | 200 | ?   | 150 | 500   |
| 2020 | 250 | 180 | 200 | 630   |

Caselet: "Company B's revenue in 2019 was 30% less than its revenue in 2020."

FIND B_2019:
B_2020 = 180. B_2019 = 180 × (1 − 0.30) = 180 × 0.70 = 126.

VERIFY with row total: 200+126+150 = 476 ≠ 500. Contradiction!

ALTERNATE APPROACH — Use row total directly:
B_2019 = 500 − 200 − 150 = 150.

Check against caselet: 150 should be 30% less than 180.
150 / 180 = 0.833 → 150 is 16.67% less than 180 (not 30%). Inconsistency!

In such cases: The chart's "Total" column gives a more reliable clue.
Use the row total clue first (it's more direct), then check if caselet is consistent.

IN A WELL-DESIGNED EXAM QUESTION: Both clues will be consistent.
Both will give B_2019 = 150. If they don't, there may be a typo in the question.
Always prefer the most direct numerical clue.
```

## 12. CONCEPT 8 — TWO MISSING VALUES (SYSTEM OF EQUATIONS)

---

### When Two Missing Values Require Each Other

```
SCENARIO: Two cells are missing in the same row AND same column.
The row total AND the column total are both given.
Neither missing value can be found without knowing the other.

SOLUTION: Set up a system of equations.

EXAMPLE:
| Year | A  | B  | C  | Row Total |
|------|----|----|----|----|
| 2018 | 200|  x | 150|   500     |
| 2019 | 180|  y | 200|   620     |
| Col B total = 350 |

Equation 1 (Row 2018): 200 + x + 150 = 500 → x = 150.
Equation 2 (Row 2019): 180 + y + 200 = 620 → y = 240.
Equation 3 (Col B total): x + y = 350 → 150 + 240 = 390 ≠ 350. Contradiction!

In a WELL-DESIGNED QUESTION: All three equations are consistent.
If they're not → exam likely has only 2 out of the 3 constraints as given.

CORRECT SETUP (consistent example):
| Year | A  | B  | C  | Row Total |
|------|----|----|----|----|
| 2018 | 200|  x | 100|   450     |
| 2019 | 180|  y | 120|   ???     |
| Col B total = 300 |

From Row 2018: x = 450 − 200 − 100 = 150.
From Col B total: y = 300 − 150 = 150.
Row 2019 total = 180 + 150 + 120 = 450 (derivable, not given as separate clue).

ALWAYS: Find the one that CAN be computed directly. Then use it for the other.
```

---

### True System of Equations (Both Truly Interdependent)

```
RARE SCENARIO: Neither missing value can be found from a single constraint alone.
TWO constraints TOGETHER determine both.

Example:
Missing values: x and y.
Constraint 1: x + y = 250.
Constraint 2: x − y = 50.

Solve: 2x = 300 → x = 150. y = 100.

This requires SIMULTANEOUS EQUATIONS, which is unusual in DI but appears in advanced exams.

WHEN TO RECOGNIZE: If neither constraint alone pins down one variable → must solve simultaneously.
```

## 13. MASTER TRICKS & SHORTCUTS

---

### 🚀 TRICK 1 — FIND ALL Missing Values Before Question 1 (Non-Negotiable)

```
This is the cardinal rule of Missing DI.

HABIT: Before reading Question 1, scan the ENTIRE chart.
       Find every "?", blank, or absent element.
       Compute all missing values using available clues.
       Write down every missing value clearly.

TIME INVESTED: 1–3 minutes.
BENEFIT: Answering each of 5 questions takes < 30 seconds.

If you skip this step:
→ Q1 asks about the missing value → you're stuck.
→ Q3 uses the missing value indirectly → you get it wrong.
→ Q5 asks for total including missing value → wrong answer.

NEVER skip the FIND step.
```

---

### 🚀 TRICK 2 — Row Total Subtraction Formula

```
The most common clue in Missing DI (tables):
Missing cell = Row total − Sum of all other cells in that row.

MENTAL MATH SHORTCUT:
Instead of adding all known cells then subtracting:
→ Start with the row total.
→ Subtract each known cell one by one.

Example: Row total = 900. Known: 200, 150, 250, 130.
900 − 200 = 700.
700 − 150 = 550.
550 − 250 = 300.
300 − 130 = 170. ← Missing value.

This sequential subtraction is faster than: (200+150+250+130=730, then 900−730=170).
```

---

### 🚀 TRICK 3 — Average Clue: Total First, Then Subtract

```
When average is given:
Step 1: Compute total = Average × Count.
Step 2: Missing = Total − Sum of known values.

MENTAL MATH:
Average × Count can be computed mentally if average is a round number:
Average = 60, Count = 5: Total = 300.
Average = 120, Count = 4: Total = 480.
Average = 250, Count = 6: Total = 1500.

Memorize: Average × Count = Total. Always.
```

---

### 🚀 TRICK 4 — Pie Chart Missing Sector by Subtraction

```
ALL pie chart sectors MUST sum to 100% (or 360°).
Missing % = 100% − (Sum of all given %).

This is the ONLY formula you ever need for missing pie sectors.
No other clue is needed.

INSTANT COMPUTATION:
Add all given percentages mentally:
30+25+15+10 = 80. Missing = 100−80 = 20%.

For degrees:
120+90+72+36 = 318. Missing = 360−318 = 42°.

Never needs more than 10 seconds once you identify the missing sector.
```

---

### 🚀 TRICK 5 — The "Scan Before You Answer" Rule

```
In Missing DI, every question has a TRAP:
Some questions seem straightforward but use the missing value indirectly.

Example:
Q: "What is the total production of all companies in 2019?"
This requires summing ALL companies in 2019, INCLUDING the missing one.
If you haven't found the missing value, you can't answer this.

SCAN EVERY QUESTION: Does this question involve the missing cell/bar/sector?
If YES → ensure you've found the missing value before computing.
If NO → proceed directly without needing the missing value.

This prevents "accidental omission" of the missing value in totals.
```

---

### 🚀 TRICK 6 — Cross-Verification Using Two Clues

```
When two independent clues are given for two different missing values,
ALWAYS cross-verify using a third constraint if available.

Example:
MV1 found using Row Total = 220.
MV2 found using Col Total = 350.

Third constraint: Grand Total of entire table = 1500.
Check: Sum of all known cells + MV1 + MV2 = 1500?
If yes → both values verified.
If no → one of them is wrong. Recompute.

This SAVES you from losing 4–5 marks due to a single arithmetic error.
```

---

### 🚀 TRICK 7 — % Change Clue: Use Multiplier Directly

```
When the clue is "missing year is X% more/less than known year":
Use the multiplier directly.

+X% more: Missing = Known × (1 + X/100).
−X% less: Missing = Known × (1 − X/100).

Quick multipliers:
+20%: ×1.20 | +25%: ×1.25 | +50%: ×1.50
−20%: ×0.80 | −25%: ×0.75 | −50%: ×0.50

Example: 2019 = 250. 2020 = 25% more.
2020 = 250 × 1.25 = 312.5.

This avoids the two-step: "25% of 250 = 62.5, then 250+62.5 = 312.5."
One multiplication is faster.
```

---

### 🚀 TRICK 8 — Ratio Clue: Multiply Known by Ratio

```
When clue is "missing : known = m : n":
Missing = Known × (m/n).

When clue is "known : missing = m : n":
Missing = Known × (n/m).

Example: 2019 : 2020 = 3 : 4. 2019 = 180.
2020 (missing) = 180 × (4/3) = 240.

INSTANT CHECK: Is the ratio larger or smaller?
4/3 > 1 → missing is larger than known. 240 > 180. ✓
```

---

### 🚀 TRICK 9 — Identify Clue Location Quickly

```
Clues in Missing DI appear in these locations:
1. The "Total" row at the bottom of the table.
2. The "Total" column at the right of the table.
3. A note below the chart: "All totals are as shown."
4. The question stem: "If total = 1200, find..."
5. A caselet/paragraph above or below the chart.
6. The chart title: "Average = 500 units per year."

SCAN LOCATIONS IN THIS ORDER:
1. Bottom row and right column (most common for tables).
2. Chart title and subtitle.
3. Notes below the chart.
4. All question stems (scan quickly before answering).

Finding the clue fast = finding the missing value fast = answering questions fast.
```

---

### 🚀 TRICK 10 — Eliminate Answer Choices Using Missing Value Sign

```
After finding the missing value, use its sign/magnitude to quickly eliminate answer choices.

Example:
Missing value = 120.
Q: "What % is the missing company's sales of total sales?"

If total = 1200: % = 120/1200 × 100 = 10%.
Options: 8%, 10%, 12%, 15%.
Answer: 10%.

You can verify without full computation:
120/1200 = 1/10 = 10%. One-step fraction. ✓

Similarly, if missing value is the largest or smallest, quickly identify max/min questions.
```

---

### 🚀 TRICK 11 — Grand Total Verification

```
After finding ALL missing values, compute the grand total of the table
and verify against any given grand total.

Grand total = Sum of ALL row totals = Sum of ALL column totals.
Include missing values in this sum.

If your computed grand total matches the given one → ALL missing values correct.
If not → at least one missing value is wrong. Find the discrepancy.

This single check verifies ALL your work at once.
```

---

### 🚀 TRICK 12 — "Questions Reveal Clues" Strategy

```
When you cannot find the clue for a missing value from the chart/notes:
SCAN ALL QUESTION STEMS before solving any question.

Often, Q3 or Q4 contains: "If the [missing entity]'s value = X, find..."
This X IS the missing value (or gives you the clue to compute it).

Example:
Q3: "If Company B's revenue in 2019 was ₹180 Cr, then what % of total was it?"
The number ₹180 Cr IS the missing value. Use it for all other questions too.

STRATEGY:
After finding all chart-based clues, scan all 5 question stems.
If any question gives a specific number for the missing entity → that's your missing value.
```

---

### 🚀 TRICK 13 — Dependency Chain for Multiple Missing Values

```
When 3+ values are missing, draw a dependency chain:

MV1 (independent: found from row total 1)
   ↓ used in
MV2 (found from column total, uses MV1)
   ↓ used in
MV3 (found from grand total, uses MV1 and MV2)

Always compute TOP DOWN in the dependency chain.
Never try to find MV3 before MV1 and MV2.

VISUAL STEP: Before computing, write:
"Order of computation: MV1 → MV2 → MV3."
Then compute in that exact order.
```

---

### 🚀 TRICK 14 — Quick Fraction for "% Share" Questions After Missing Value Found

```
After finding missing value (MV), many questions ask:
"MV is what % of total?" or "MV is what % of row total?"

Use the fraction shortcut:
% = (MV / Total) × 100.

If Total = 500: % = MV/5. (Divide by 5 instead of computing ×100/500.)
If Total = 1000: % = MV/10.
If Total = 200: % = MV/2.
If Total = 2000: % = MV/20.
If Total = 400: % = MV/4.

These "friendly total" shortcuts eliminate the need for long division.
```

---

### 🚀 TRICK 15 — Sanity Check on Missing Values

```
After computing a missing value, apply these sanity checks:

1. Is it POSITIVE? (Revenue, production, employees cannot be negative.)
2. Is it within a REASONABLE RANGE relative to other values?
   (If other bars are 100–300, a missing value of 5000 is suspicious.)
3. Does it sum correctly? (Missing + known = row/column total.)
4. Does it make logical sense? (A company can't have negative employees.)

If the missing value fails any sanity check:
→ Recheck your arithmetic.
→ Recheck which clue you used.
→ Recheck which cells are "other known values."

A negative or extremely outlying missing value almost always signals an arithmetic error.
```

## 15. COMMON MISTAKE ALERTS

---

### ⚠️ MISTAKE 1 — Not Finding All Missing Values Before Answering

```
Wrong: Reading Q1 and trying to answer without finding the missing value first.
Right: FIND all missing values → then answer all questions.

Prevention: Make it a non-negotiable habit.
            Before Question 1, look at the chart and find every "?".
```

---

### ⚠️ MISTAKE 2 — Using Row Total for Column or Vice Versa

```
Wrong: Using the column B total to find a missing value in ROW 2019.
Right: Missing in row 2019 → use ROW 2019 total.
       Missing in column B → use COLUMN B total.

Direction matters completely.
Row total: sum across a row. Column total: sum down a column.

Prevention: Explicitly confirm: "Is this missing value in a ROW or a COLUMN?"
            Then use the appropriate total.
```

---

### ⚠️ MISTAKE 3 — Arithmetic Error in Finding Missing Value

```
If you make an arithmetic error when finding the missing value:
EVERY question that uses that value will be wrong.
This can cost 4–5 marks from one computation mistake.

Prevention:
1. Compute slowly and carefully for the missing value step.
2. Cross-verify using a second clue if available.
3. Apply the sanity check (positive? reasonable range?).
```

---

### ⚠️ MISTAKE 4 — Including Missing Value in Sum Before Finding It

```
Wrong: "Total 2019 = A + ? + C + D = X." (Adding ? into the sum without knowing it.)
Right: Find ? first. Then compute the total using the actual value.

Prevention: When computing a sum for a row/column that CONTAINS a missing value:
            → Either use the row/column total directly.
            → Or find the missing value first, THEN compute the sum.
```

---

### ⚠️ MISTAKE 5 — Wrong Order for Dependent Missing Values

```
Wrong: Trying to find MV2 (which needs MV1) before finding MV1.
Right: Find MV1 first (independent). Then use MV1 to find MV2.

Prevention: Before computing, determine which missing values are independent
            and which are dependent. Compute in dependency order.
```

---

### ⚠️ MISTAKE 6 — Pie Chart: Forgetting That All Sectors Must Sum to 100%

```
Wrong: Computing the missing sector as "the given total minus something."
Right: Missing % = 100% − Sum of all given %.

This is the ONLY formula for missing pie sectors.
No matter how many sectors or what the total is, the % sum is ALWAYS 100%.

Prevention: Immediately sum all given percentages and subtract from 100.
```

---

### ⚠️ MISTAKE 7 — % Change Direction Error

```
Wrong: "2020 is 20% more than 2019 (=200)" → 2020 = 200 × 0.80 = 160 (wrong — that's 20% LESS).
Right: 20% MORE → multiply by 1.20. 2020 = 200 × 1.20 = 240.

Prevention:
+X% more → multiply by (1 + X/100) — multiplier > 1.
−X% less → multiply by (1 − X/100) — multiplier < 1.

Check: Is missing > known? If "more" → YES. Is missing < known? If "less" → YES.
```

## 16. SOLVED EXAMPLES (BASIC TO ADVANCED)

### 📘 EXAMPLE SET 1 — MISSING VALUE IN TABLE (BASIC)

**Table: Production of 4 Companies (in lakh units)**

| Year | A   | B   | C   | D   | Row Total |
|------|-----|-----|-----|-----|-----------|
| 2018 | 200 | 150 | 180 |  ?  |    620    |
| 2019 | 240 | 180 |  ?  | 160 |    760    |
| 2020 | 280 | 200 | 220 | 190 |    890    |
| 2021 | 320 |  ?  | 250 | 210 |   1040    |
| 2022 | 360 | 280 | 280 | 230 |   1150    |

---

**FIND ALL MISSING VALUES:**

```
MV1 (2018, D): D = 620 − 200 − 150 − 180 = 620 − 530 = 90.
MV2 (2019, C): C = 760 − 240 − 180 − 160 = 760 − 580 = 180.
MV3 (2021, B): B = 1040 − 320 − 250 − 210 = 1040 − 780 = 260.

COMPLETE TABLE:
| Year | A   | B   | C   | D   | Total |
|------|-----|-----|-----|-----|-------|
| 2018 | 200 | 150 | 180 |  90 |  620  |
| 2019 | 240 | 180 | 180 | 160 |  760  |
| 2020 | 280 | 200 | 220 | 190 |  890  |
| 2021 | 320 | 260 | 250 | 210 | 1040  |
| 2022 | 360 | 280 | 280 | 230 | 1150  |

VERIFY MV1: 200+150+180+90=620 ✓
VERIFY MV2: 240+180+180+160=760 ✓
VERIFY MV3: 320+260+250+210=1040 ✓
```

**Q1. What is the total production of Company B over all 5 years?**
```
150+180+200+260+280 = 1070 lakh units.
```

**Q2. By what % did Company C's production increase from 2018 to 2022?**
```
Old = 180 (2018), New = 280 (2022).
% increase = [(280−180)/180] × 100 = (100/180) × 100 = 55.56%.
```

**Q3. D's production in 2018 is what % of D's total production over all 5 years?**
```
D total = 90+160+190+210+230 = 880.
% = (90/880) × 100 = 10.23%.
```

**Q4. In 2021, which company had the highest production?**
```
A=320, B=260, C=250, D=210. Maximum = A (320 lakh units).
```

**Q5. What is the ratio of Company A's total production to Company D's total production?**
```
A total = 200+240+280+320+360 = 1400.
D total = 90+160+190+210+230 = 880.
Ratio = 1400:880 = 35:22 (HCF=40).
```

DERIVED PROFITS:

**Q2. In which year was Company B's profit margin maximum?**

B profit margins:

## 🏆 TOP 15 EXAM TIPS

1. **Find ALL missing values before answering Question 1 — without exception** — this is the cardinal rule of Missing DI. Investing 1–3 minutes finding all missing values pays back 5× in question-answering speed. Skipping this step causes every answer that depends on the missing value to be wrong.

2. **The missing value is never truly absent** — it's always computable from a clue somewhere in the DI set. If you can't find the clue, check: the total row, total column, chart title, footnote, and all five question stems. The clue is always there.

3. **Use row total subtraction sequentially** — instead of summing known values then subtracting (two steps), subtract each known value from the row total directly (one continuous process). Faster and less error-prone.

4. **For pie charts: missing % = 100% − sum of all given %** — this is the only formula you need. All sectors must sum to 100%. This takes under 10 seconds. Never attempt any other approach for a missing pie sector.

5. **Compute dependencies in the right order** — if MV2 requires MV1 to be known, you must find MV1 first. Before computing anything, determine which missing values are independent (findable directly) and which are dependent (need another MV first).

6. **Cross-verify every missing value when a second clue is available** — if a missing cell is in both a row (row total given) and a column (column total given), use one clue to find the value and the second to verify. This catches arithmetic errors before they cascade into wrong answers.

7. **Apply the sanity check immediately** — after computing a missing value, ask: Is it positive? Is it in a reasonable range compared to adjacent values? Does it sum correctly with the others? A negative or wildly outlying value signals a computation error.

8. **For % change clues: use the multiplier, not the two-step approach** — "20% more than X" → multiply X by 1.20 directly. Don't compute 20% of X separately and then add. One multiplication is faster and has fewer opportunities for error.

9. **Scan all question stems before solving, especially when the clue is missing** — sometimes the clue for the missing value is embedded in a specific question ("If Company B's 2019 revenue = ₹180 Cr..."). Scan all questions upfront to find such embedded clues.

10. **Grand total verification = instant check for ALL missing values** — after finding all missing values, compute the grand total of the table (or total of all bars/sectors) and verify it matches any given grand total. If it does, ALL your missing values are correct.

11. **For two missing values: look for one that is independent** — in most multi-missing-value tables, one missing value can be found from a row total, and the other from a column total (which now includes the first). Find the independent one first.

12. **"Questions reveal clues" when the chart clue is insufficient** — if you've exhausted all chart-based clues and still can't find a missing value, read all question stems. Some questions contain the missing value explicitly ("given that [missing entity] = X, find...").

13. **Use friendly total shortcuts for quick % calculations** — after finding a missing value, many questions ask "what % is this of the total?" If total = 1000, just divide by 10. If total = 500, divide by 5. These shortcuts eliminate long division for the most common totals.

14. **A consistent second clue = maximum confidence** — if two different clues (e.g., row total AND an average clue) both give you the same missing value, you can answer all 5–8 questions with complete confidence. Inconsistency → re-examine the clue you used.

15. **Practice finding missing values in 60 seconds** — the competitive advantage in Missing DI is speed of the "FIND" step. Exam toppers find all missing values in under 60 seconds and spend the remaining time answering questions. Target: 60 seconds for FIND, 20–30 seconds per question.

---

*Missing DI is nothing more than regular DI with one extra step: finding the hidden value before anything else. The DI formulas (ratio, %, average, difference) are identical to what you've learned in Bar Graph, Pie Chart, Table, and Line Graph DI. The unique skill is the FIND step — locating the clue, computing the missing value, and verifying it. Master the FIND method, the 7 clue locations, and the dependency order rule — and Missing DI will become one of your most reliable scoring sections. You've got this! 💪*
