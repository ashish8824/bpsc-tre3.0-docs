## 1. WHAT IS A BAR GRAPH?

A **Bar Graph** (also called a **Bar Chart**) is a pictorial representation of data using **rectangular bars**.

- Each bar represents a **category** (year, city, product, department, etc.)
- The **height (or length)** of the bar shows the **value / quantity** of that category
- Bars are always **separated by gaps** — this distinguishes a bar graph from a histogram
- Bars can be **vertical** (most common) or **horizontal**

---

### Why It Is Tested

```
Bar Graphs test your ability to:
  ├── Read and extract values accurately from a visual scale
  ├── Perform arithmetic quickly (difference, ratio, %, average)
  ├── Compare values across categories or time periods
  ├── Identify trends (increasing, decreasing, fluctuating)
  ├── Derive hidden values (profit = sales − expenditure, etc.)
  └── Avoid visual traps (scale not starting at 0, misleading sizes)
```

---

### The Two Core Challenges of Every Bar Graph Question

```
CHALLENGE 1 — READING: Did you extract the right value from the right bar?
CHALLENGE 2 — CALCULATING: Did you apply the correct formula to those values?

A correct reading + wrong formula = WRONG.
A correct formula + misread bar = WRONG.
Both must be right.
```

## 2. TYPES OF BAR GRAPH QUESTIONS

---

### Type 1 — Direct Value Reading (Easiest)

Read the height of a bar and report it.

```
"What was the production of wheat in the year 2019?"
→ Find the 2019 bar → read its height against the Y-axis → done.

Trap: Misreading the scale. If each gridline = 5 units and the bar is halfway
between 40 and 45, the value is 42.5 — not 40, not 45.
```

---

### Type 2 — Maximum / Minimum Identification

Identify which bar is tallest (max) or shortest (min).

```
"In which year was production maximum?"
→ Scan all bars visually → tallest bar = answer.
→ DO NOT calculate unless two bars look almost equal.

Trap: In a double bar graph, the question might ask for
"maximum production of Product A only" — don't accidentally
compare across both products.
```

---

### Type 3 — Difference Between Values

```
"What is the difference in sales between 2020 and 2022?"
→ Larger value − Smaller value → answer.
```

---

### Type 4 — Ratio

```
"Find the ratio of exports in 2018 to exports in 2021."
→ Value A : Value B → simplify by HCF → answer.
```

---

### Type 5 — Percentage Change (Increase / Decrease)

```
"By what % did production increase from 2018 to 2022?"
→ % Change = [(New − Old) / Old] × 100
→ Positive = increase | Negative = decrease
```

---

### Type 6 — Average

```
"What is the average production over the given 5 years?"
→ Average = Sum of all values ÷ Number of values
```

---

### Type 7 — Percentage Share

```
"Production in 2019 is what % of total production over all years?"
→ (Value of 2019 / Total of all years) × 100
```

---

### Type 8 — What % more / less

```
"Production in 2021 is what % more than production in 2019?"
→ [(2021 − 2019) / 2019] × 100
```

---

### Type 9 — Derived Value (Profit, Balance, Savings)

```
"In which year was profit (= Sales − Expenditure) maximum?"
→ Calculate profit = Sales bar − Expenditure bar for each year
→ Identify the year with highest resulting value
```

---

### Type 10 — Projection / Estimation

```
"If production in 2023 is expected to be 15% more than 2022, what will it be?"
→ 2022 value × 1.15 = 2023 projected value
```

## 4. CONCEPT 1 — READING A BAR GRAPH

> The most important skill in bar graph DI is **accurate reading**. A calculation error costs time. A reading error costs the entire mark — and often leads to a confidently wrong answer.

---

### 4.1 The Five Reading Layers

**Every time you encounter a new bar graph, read these five layers BEFORE looking at any question.**

---

#### LAYER 1 — Title

Read the title in full. It tells you:
- What is being measured (production? sales? population? exports?)
- The unit (in lakh tonnes, in ₹ crores, in thousands, etc.)
- The scope (which years? which countries? which departments?)

```
Example title: "Production of Steel (in lakh metric tonnes) from 2015 to 2022"
→ Subject: Steel production
→ Unit: lakh metric tonnes — every bar value must be read in this unit
→ Scope: 2015 to 2022
```

---

#### LAYER 2 — X-axis

The horizontal axis. Usually shows **categories** — years, months, cities, products, companies.

```
Always note: How many categories are there?
Are they time-based (trend) or category-based (comparison)?
```

---

#### LAYER 3 — Y-axis and Scale

The vertical axis. Shows **values**. This is where most reading errors occur.

```
Always note:
1. Does the Y-axis start at 0 or at some other value?
   (If it starts at 50, a bar reaching 60 does NOT represent double a bar reaching 55)
2. What is each gridline interval?
   (If gridlines are at 0, 20, 40, 60... each interval = 20)
3. What is each small division worth?
   (If there are 4 small divisions between 0 and 20, each = 5)
```

---

#### LAYER 4 — Legend

In double/multiple bar graphs and stacked bar graphs, a legend (key) appears.

```
Always match: Which colour / pattern / shading = which variable?
Mark it mentally: "Dark bar = Sales | Light bar = Expenditure"
Misreading the legend is a silent killer — you get a confident answer that's completely wrong.
```

---

#### LAYER 5 — Trend

Scan the bars for overall direction. This helps you sanity-check your answers.

```
If bars are generally increasing year-on-year, and you calculate a large decrease
for some year — re-check before submitting.
```

---

### 4.2 Scale Reading — The Most Common Error Source

```
CASE 1: Y-axis gridlines every 10 units
If a bar touches the 3rd gridline → value = 3 × 10 = 30

CASE 2: Y-axis gridlines every 20 units, bar is BETWEEN gridlines
Gridlines at 40 and 60. Bar is halfway between = 50.
Bar is 1/4 of the way from 40 to 60 = 40 + 5 = 45.

CASE 3: Y-axis does NOT start at zero
If Y-axis starts at 500 and goes in steps of 100:
A bar at the 3rd gridline = 500 + 3×100 = 800
(NOT 300, which you'd get if you assumed it started at 0)

CASE 4: Scale given in lakhs, question asks in crores
1 crore = 100 lakhs
Read the bar in lakhs, divide by 100 to convert to crores.
```

---

### 4.3 Reading Double / Multiple Bar Graphs

```
Step 1: Confirm from the legend which bar is which variable.
Step 2: For each category on the X-axis, there are two (or more) bars side by side.
Step 3: Read each bar separately — they share the same Y-axis scale.
Step 4: When comparing, only compare the SAME variable across categories,
        OR explicitly compare the two variables for the SAME category.

Common confusion: Accidentally reading the "Sales" bar when the question asks for "Profit."
Prevention: Underline / mentally highlight which variable you need before reading.
```

## 5. CONCEPT 2 — TYPES OF BAR GRAPHS

---

### 5.1 Simple Bar Graph

One bar per category. Represents a **single variable**.

```
Example: Monthly revenue of a company

Month  | Jan | Feb | Mar | Apr | May | Jun
Revenue|  40 |  55 |  50 |  70 |  65 |  80
(₹ lakhs)

   80 |                              ████
   70 |                    ████      ████
   60 |                    ████ ████ ████
   50 |        ████  ████  ████ ████ ████
   40 | ████   ████  ████  ████ ████ ████
      +------+-----+-----+----+----+----
        Jan   Feb   Mar   Apr  May  Jun

→ Reading: Each bar's height = that month's revenue in ₹ lakhs.
→ Questions will ask for max, min, average, ratio, % change.
```

---

### 5.2 Double / Multiple Bar Graph

Two or more bars **side by side** for each category. Compares multiple variables.

```
Example: Sales (S) and Expenditure (E) of a company

Year  | 2019 | 2020 | 2021 | 2022
Sales |  100 |   90 |  150 |  180
Exp.  |   80 |   85 |  100 |  120

Each year has TWO bars — one for Sales (dark), one for Expenditure (light).

→ Derived variable: Profit = Sales − Expenditure
→ Questions will ask: "In which year was profit maximum?" (requires subtraction for each year)
→ Trap: "Sales in 2020 is what % more than Expenditure in 2020?" → same year, two variables
         vs "Sales in 2020 is what % more than Sales in 2019?" → same variable, two years
         These use DIFFERENT formulas for the base.
```

> ⚡ Most exam DI sets use double bar graphs. Mastering them = cracking most DI questions.

---

### 5.3 Stacked Bar Graph (Composite)

One bar per category, but the bar is **divided into segments**, each representing a sub-category.

```
Example: Total production divided into Product A and Product B

Year   | 2020 | 2021 | 2022
Prod A |   60 |   80 |  100
Prod B |   40 |   50 |   70
Total  |  100 |  130 |  170

In the graph, the 2020 bar reaches height 100.
The bottom segment (0 to 60) = Product A = 60
The top segment (60 to 100) = Product B = 40

→ Total = height of entire bar
→ Segment value = Top of segment − Bottom of segment
→ DO NOT add bars separately — read the full bar height for the total.
```

---

### 5.4 Percentage Stacked Bar Graph

Like the stacked bar graph, but every bar is exactly **100% tall**. Segments show **percentage share**.

```
Example:
Year   | Share of Product A | Share of Product B
2020   |        60%         |        40%
2021   |        62%         |        38%
2022   |        59%         |        41%

→ The actual values are NOT directly given — only proportions.
→ To find actual value, you need total + percentage.
→ Questions: "Which year did Product B have the highest share?" → visual comparison.
```

---

### 5.5 Horizontal Bar Graph

Same as a simple bar graph but **rotated 90°**. Bars run left to right. Values are on the X-axis; categories on the Y-axis.

```
→ Used when category names are long (e.g., country names, company names).
→ All formulas are identical. The rotation changes nothing about the calculations.
→ Reading: bar LENGTH (horizontal) = value, not bar height.
```

## 6. CONCEPT 3 — CORE CALCULATIONS

> Every bar graph question uses one (or a combination) of these seven calculation types. Master each formula with its exact base and application.

---

### 6.1 Difference

The simplest calculation. Subtract one value from another.

```
Formula: Difference = Larger value − Smaller value

Example:
Production in 2022 = 80 units
Production in 2018 = 50 units
Difference = 80 − 50 = 30 units

Trap: The question says "how much MORE" — answer is the difference, NOT a percentage.
      "By what % more" — that requires the % formula (Section 6.6).
```

---

### 6.2 Ratio

Express two values as a simplified fraction.

```
Formula: Ratio of A to B = A : B
         Simplify both sides by dividing by their HCF.

Example:
Sales in 2019 = 120 | Sales in 2021 = 180
Ratio = 120 : 180
HCF of 120 and 180 = 60
Simplified ratio = 2 : 3

Quick HCF trick: Divide both by the obvious common factor.
120 and 180 → both divisible by 60 → 2:3
If not obvious, try 2, then 3, then 5, then 10.
```

---

### 6.3 Average / Mean

```
Formula: Average = Sum of all values ÷ Number of values

Example:
Production across 5 years: 40, 60, 50, 80, 70
Sum = 40 + 60 + 50 + 80 + 70 = 300
Average = 300 ÷ 5 = 60

The Deviation Method (faster when values are close):
Choose a base near the middle (say 60).
Deviations: −20, 0, −10, +20, +10
Sum of deviations = 0
Average = 60 + (0/5) = 60 ✓

Another example:
Values: 48, 52, 55, 49, 51 → Base = 50
Deviations: −2, +2, +5, −1, +1 → Sum = +5
Average = 50 + (5/5) = 50 + 1 = 51 ✓ (Much faster than adding 48+52+55+49+51)
```

---

### 6.4 Percentage Change (Increase / Decrease)

> The most tested formula in bar graph DI. Know it perfectly.

```
Formula: % Change = [(New Value − Old Value) / Old Value] × 100

Positive result = % Increase
Negative result = % Decrease (report the magnitude without the minus sign)

CRITICAL: The base is ALWAYS the OLD (earlier / original) value — not the new one.

Example 1 (Increase):
Sales 2020 = 200 | Sales 2021 = 250
% Change = [(250 − 200) / 200] × 100 = [50/200] × 100 = 25% increase

Example 2 (Decrease):
Production 2021 = 300 | Production 2022 = 270
% Change = [(270 − 300) / 300] × 100 = [−30/300] × 100 = −10% → 10% decrease

Trap: Never use the NEW value as the base. This is the single most common
error in percentage change questions.
```

---

### 6.5 Percentage Share

```
Formula: % Share of item X = (Value of X / Total of all items) × 100

Example:
Production in 5 years: 40, 60, 50, 80, 70 → Total = 300
% share of 2021 (value = 80) = (80/300) × 100 = 26.67%

Quick simplification: 80/300 = 8/30 = 4/15
4/15 × 100 = 400/15 = 26.67%
```

---

### 6.6 "What % more / less" vs "What % of"

> These two formulas are completely different. The most common trap in DI exams.

```
"A is what % OF B?"
Formula: (A/B) × 100
→ This gives the proportion of A relative to B.

"A is what % MORE THAN B?"
Formula: [(A − B) / B] × 100
→ This gives the excess of A over B, as a percentage of B.

"A is what % LESS THAN B?"
Formula: [(B − A) / B] × 100
→ This gives how much A falls short of B, as a percentage of B.

CRUCIAL RULE: In "more than" and "less than" questions,
the BASE is always the REFERENCE VALUE (the "than" value = B).

Numerical Example (A = 120, B = 100):
"A is what % of B?" → (120/100) × 100 = 120%
"A is what % more than B?" → [(120−100)/100] × 100 = 20%
"B is what % less than A?" → [(120−100)/120] × 100 = 16.67%

Note: 20% MORE ≠ 20% LESS — the bases are different!
20% more (base B=100) vs 16.67% less (base A=120).
```

---

### 6.7 Profit, Savings, Trade Balance — Derived Values

Many bar graph questions involve a variable that is NOT directly shown in the graph, but must be **derived** from two visible variables.

```
Common Derived Values:
  Profit / Loss = Sales − Expenditure
  Savings       = Income − Expenditure
  Trade Balance = Exports − Imports
    (positive = Trade Surplus | negative = Trade Deficit)
  Net Production = Total Production − Losses / Waste
  Per Capita     = Total Value ÷ Population

For derived value questions:
Step 1: Calculate the derived value for EACH relevant year / category.
Step 2: Apply the required formula (max, %, ratio, etc.) to the derived values.

Example:
Year  | Sales | Expenditure | Profit (derived)
2019  | 100   | 80          | 20
2020  | 90    | 85          | 5
2021  | 150   | 100         | 50
2022  | 180   | 120         | 60

"In which year was % profit on sales maximum?"
% Profit = (Profit / Sales) × 100
2019: (20/100) × 100 = 20%
2020: (5/90) × 100  = 5.6%
2021: (50/150) × 100 = 33.3%
2022: (60/180) × 100 = 33.3%
→ Both 2021 and 2022 are tied at 33.3%
```

## 7. CONCEPT 4 — FINDING MAXIMUM PERCENTAGE CHANGE

> In many questions you are asked: "In which year was the percentage increase maximum?" Calculating % change for every pair of years is slow. Use this visual trick instead.

---

### The Visual Comparison Method

```
% Increase from Year N to Year N+1 = (Rise / Previous bar height) × 100

For the % to be MAX:
→ You want a LARGE RISE combined with a SMALL PREVIOUS BAR.
→ A large bar rising a little = small % change.
→ A small bar rising a lot = large % change.

Example:
Year  | Value | Rise from prev. | % Change
2018  | 200   | —               | —
2019  | 240   | 40              | (40/200)×100 = 20%
2020  | 300   | 60              | (60/240)×100 = 25%
2021  | 330   | 30              | (30/300)×100 = 10%
2022  | 360   | 30              | (30/330)×100 = 9.1%

→ Maximum % change: 2019→2020 (25%)
→ Visual shortcut: 2019→2020 has a good rise (60) on a moderate base (240).
  2021→2022 has the same rise (30) but a larger base (330) → smaller %.
```

---

### The Fraction Comparison Shortcut

```
Instead of computing each % exactly, compare the fractions:
40/200 vs 60/240 vs 30/300 vs 30/330

Cross-multiply pairs to compare:
40/200 vs 60/240 → 40×240=9600 vs 60×200=12000 → 12000>9600 → 60/240 is larger
60/240 vs 30/300 → 60×300=18000 vs 30×240=7200 → 18000>7200 → 60/240 is larger
→ Maximum is 60/240 → Year 2019→2020. ✓

This avoids all division — pure multiplication comparison.
```

## 8. CONCEPT 5 — STACKED BAR CALCULATIONS

---

### Reading Segment Values from a Stacked Bar

```
In a stacked bar graph, bars are divided into colour-coded segments.

Reading Rule:
→ Total value of the bar = Height of the entire bar (read against Y-axis)
→ Value of BOTTOM segment = Height of bottom segment (read from 0 to top of segment)
→ Value of TOP segment = Total bar height − Bottom segment height
→ Value of MIDDLE segment (if 3 segments) = Top of that segment − Bottom of that segment

Example:
A bar has total height 180.
Bottom segment (A) ends at 80 → Value of A = 80
Middle segment (B) ends at 130 → Value of B = 130 − 80 = 50
Top segment (C) ends at 180 → Value of C = 180 − 130 = 50
Total check: 80 + 50 + 50 = 180 ✓
```

---

### % Share of a Segment

```
% share of segment A = (Value of A / Total bar height) × 100

Example: A = 80, Total = 180
% share of A = (80/180) × 100 = 44.4%
```

---

### Ratio of Two Segments

```
Ratio of A to B = A : B = 80 : 50 = 8 : 5

Or across two years:
"Ratio of segment A in 2021 to segment A in 2022"
→ Read A's value from each year's bar → form the ratio → simplify.
```

## 9. CONCEPT 6 — PROJECTION / ESTIMATION QUESTIONS

> These questions ask you to compute a future (or hypothetical) value based on a given percentage increase or decrease.

---

### Multiplier Method — The Fastest Approach

```
Increase by x%  → multiply by (1 + x/100)   = (100+x)/100
Decrease by x%  → multiply by (1 − x/100)   = (100−x)/100

Common multipliers to memorise:
+5%  → ×1.05     −5%  → ×0.95
+10% → ×1.10     −10% → ×0.90
+15% → ×1.15     −15% → ×0.85
+20% → ×1.20     −20% → ×0.80
+25% → ×1.25     −25% → ×0.75
+50% → ×1.50     −50% → ×0.50

Example:
"If production in 2023 is 20% more than 2022 (which was 350),
what will production in 2023 be?"
→ 350 × 1.20 = 420

Example (combined):
"Production increases by 20% in Year 1 and then decreases by 20% in Year 2.
What is the net effect starting from 300?"
→ After Year 1: 300 × 1.20 = 360
→ After Year 2: 360 × 0.80 = 288
→ Net result: 288 (NOT 300 — +20% then −20% does NOT cancel out!)
Net change = [(288−300)/300] × 100 = −4% overall decrease
```

> ⚡ **Critical Insight:** A x% increase followed by a x% decrease always gives a net DECREASE of (x²/100)%. For 20%: net change = −(400/100)% = −4%.

## 10. CONCEPT 7 — RATIO & PROPORTION IN DI

---

### Simplifying Ratios Quickly

```
Rule: Divide both terms by their HCF.

Quick HCF Method:
→ Divide both by 2 (if both even), then by 2 again if needed.
→ Then by 3, then by 5.
→ Stop when no common factor remains.

Examples:
180 : 120 → ÷ 60 = 3:2
240 : 360 → ÷ 120 = 2:3
75 : 100  → ÷ 25 = 3:4
360 : 540 → ÷ 180 = 2:3
```

---

### Comparing Two Ratios (Which is Larger?)

```
Is A/B larger or smaller than C/D?
Cross-multiply: Compare A×D vs B×C.
If A×D > B×C → A/B > C/D
If A×D < B×C → A/B < C/D

Example: Is 7/9 > 8/11?
7×11 = 77 vs 8×9 = 72
77 > 72 → 7/9 > 8/11 ✓

No division needed at all.
```

---

### Finding a Value from a Given Ratio and Total

```
If A:B = 3:5 and Total = 800, find A and B.
A = [3/(3+5)] × 800 = (3/8) × 800 = 300
B = [5/(3+5)] × 800 = (5/8) × 800 = 500
Check: 300 + 500 = 800 ✓
```

## 11. MASTER TRICKS & SHORTCUTS

---

### 🚀 TRICK 1 — The RACE Method (Solve Any DI Set Systematically)

```
R — READ the graph (title, axes, scale, legend) BEFORE looking at questions
A — ANALYZE each question individually (what type? what formula?)
C — CALCULATE using only the values needed for that specific question
E — ELIMINATE options using approximation; verify the surviving option

Never start calculating before you've read the graph fully.
Never read the entire graph again for each question — do it once, do it right.
```

---

### 🚀 TRICK 2 — The Scale Pre-Check

```
Before answering ANY question, confirm:
1. Does the Y-axis start at 0?
   If NO → every bar value is (baseline + bar height), not just bar height.
2. What does each small division represent?
   Count the divisions between two gridlines, divide the interval by that count.

Example: Gridlines at 0 and 50, with 5 small divisions between them.
Each small division = 50 ÷ 5 = 10 units.
A bar at 3 small divisions above 50 = 50 + 30 = 80.
```

---

### 🚀 TRICK 3 — % Change: Base Is Always the Old Value

```
No matter how the question is phrased:
→ "increased FROM 2019 TO 2022" → base = 2019
→ "decreased FROM 2021 TO 2020" → base = 2021
→ "change OVER the period 2018−2022" → base = 2018
→ "growth from last year to this year" → base = last year

Memory anchor: Old = Base. New = Numerator's reference. Done.
```

---

### 🚀 TRICK 4 — "% More Than" vs "% Of" — Never Confuse These

```
IF THE QUESTION SAYS "... is what % OF ..."
→ Divide the first number by the second: (First / Second) × 100

IF THE QUESTION SAYS "... is what % MORE THAN ..."
→ Divide the DIFFERENCE by the second (the reference): [(First−Second)/Second] × 100

IF THE QUESTION SAYS "... is what % LESS THAN ..."
→ Divide the DIFFERENCE by the first (the larger/reference): [(First−Second)/First] × 100

Worked Example: A = 150, B = 100
A is what % of B?       → (150/100)×100 = 150%
A is what % more than B? → (50/100)×100 = 50%
B is what % less than A? → (50/150)×100 = 33.33%

All three are different answers for the same pair of numbers.
Identify the exact phrasing → apply the exact formula → no confusion.
```

### 🚀 TRICK 5 — The Deviation Method for Averages

```
When values are close to each other, use a reference point instead of summing all.

Steps:
1. Pick a convenient reference value (near the middle of the data).
2. Find deviations of each value from the reference (positive or negative).
3. Average = Reference + (Sum of deviations ÷ Number of values)

Example: 88, 92, 95, 89, 91
Reference = 90
Deviations: −2, +2, +5, −1, +1 → Sum = +5
Average = 90 + (5/5) = 90 + 1 = 91 ✓

Compare to: 88+92+95+89+91 = 455, 455÷5 = 91. Same answer, much faster.
```

### 🚀 TRICK 6 — Max % Change Without Full Calculation

```
To find which consecutive pair has the maximum % change:
1. Calculate Rise = New − Old for each consecutive pair.
2. Form the fraction Rise/Old for each pair.
3. Compare these fractions using cross-multiplication (no division needed).
4. The largest fraction = maximum % change.

Example:
Pair      Rise  Old   Fraction
2018→19    40   200   40/200
2019→20    60   240   60/240
2020→21    30   300   30/300

Compare 40/200 vs 60/240: 40×240=9600 vs 60×200=12000 → 60/240 is larger
Compare 60/240 vs 30/300: 60×300=18000 vs 30×240=7200 → 60/240 is larger
Winner: 2019→20 has maximum % change. ✓
```

---

### 🚀 TRICK 7 — The Multiplier Method for Projections

```
+x% → multiply by (100+x)/100
−x% → multiply by (100−x)/100

For chained changes:
Final = Initial × Multiplier1 × Multiplier2 × ...

Example: 400 increases by 25%, then decreases by 10%.
Final = 400 × 1.25 × 0.90 = 400 × 1.125 = 450

Do NOT apply % on original value each time — each % is on the NEW value.
```

### 🚀 TRICK 8 — Approximation for MCQ Elimination

```
When options are spread out (e.g., 18%, 26%, 34%, 42%):
→ Round values to the nearest 5 or 10 before calculating.
→ Substitute into the formula.
→ Check which option the approximate answer is closest to.

Example: (187 / 543) × 100 — options: 28%, 34%, 40%, 47%
Round: 190/540 ≈ 19/54 ≈ 1/2.84 ≈ 35%
Closest option = 34%. (Full calculation: 34.4% ✓)

Only use approximation when options are at least 5 units apart.
If options are close (like 33%, 34%, 35%), calculate exactly.
```

### 🚀 TRICK 9 — The Double Blank Strategy for Double Bar Graphs

```
In questions involving two variables (like Sales and Expenditure),
always identify WHICH variable the question refers to.

"In which year was the ratio of Sales to Expenditure maximum?"
→ This is NOT asking for maximum Sales, nor maximum Expenditure.
→ Calculate ratio for each year: S/E for each.
→ Compare ratios (use cross-multiplication).

Year  | Sales | Exp | S/E
2019  |  100  |  80 | 100/80 = 5/4
2020  |   90  |  85 | 90/85 = 18/17
2021  |  150  | 100 | 150/100 = 3/2
2022  |  180  | 120 | 180/120 = 3/2

3/2 (= 1.5) > 5/4 (= 1.25) > 18/17 (≈1.06)
→ Both 2021 and 2022 have ratio 3:2 — tied as maximum.
```

### 🚀 TRICK 10 — The +x% then −x% Trap

```
THIS IS A GUARANTEED EXAM TRAP. Learn it by heart.

+20% followed by −20% does NOT return to the original.
+20% then −20% = ×1.2 × ×0.8 = ×0.96 = 4% NET DECREASE.

Formula: Net % change when +x% followed by −x% = −(x²/100)%

For +20% then −20%: −(400/100)% = −4%
For +10% then −10%: −(100/100)% = −1%
For +25% then −25%: −(625/100)% = −6.25%

Example:
Production in 2020 = 500
2021: +20% → 500 × 1.2 = 600
2022: −20% → 600 × 0.8 = 480
Production in 2022 = 480 ≠ 500.
```

### 🚀 TRICK 11 — Identifying Trade Surplus/Deficit Quickly

```
Trade Surplus  = Exports > Imports → Trade Balance is POSITIVE
Trade Deficit  = Exports < Imports → Trade Balance is NEGATIVE

Quick scan: In each year, check if the Exports bar is TALLER or SHORTER than Imports bar.
Taller = Surplus | Shorter = Deficit.

Total Net Balance over multiple years = Sum of all individual Trade Balances.
If sum is positive → overall surplus. If negative → overall deficit.
```

### 🚀 TRICK 12 — Ratio Shortcut Using Cancellation

```
To form and simplify A:B quickly:
1. Write A/B as a fraction.
2. Cancel common factors from top and bottom (don't find HCF formally).
3. Result is your simplified ratio.

Example: 360 : 480
360/480 → Cancel 10 → 36/48 → Cancel 4 → 9/12 → Cancel 3 → 3/4
Ratio = 3:4

Alternative (HCF method):
360 and 480 → HCF = 120 → 360÷120 : 480÷120 = 3:4 ✓
```

### 🚀 TRICK 13 — Reading Stacked Bars: Segment = Difference of Levels

```
For any segment in a stacked bar:
Value of segment = Level where segment ENDS − Level where segment STARTS

This works for all segments (bottom, middle, top).

Bottom segment always starts at 0 (the base) → value = its top level.
Middle and top segments start at the top of the previous segment.

Example: A 3-segment bar with levels at: 0, 40, 90, 150
Segment 1 (bottom): 40 − 0 = 40
Segment 2 (middle): 90 − 40 = 50
Segment 3 (top):   150 − 90 = 60
Total: 40 + 50 + 60 = 150 ✓
```

### 🚀 TRICK 14 — Average Without Adding All Values

```
If you need the average of values that include one outlier:
Exclude the outlier → compute average of the rest → adjust.

Example: Average of 50, 55, 60, 58, 200
The 200 is an outlier. Without it: (50+55+60+58)/4 = 223/4 = 55.75
Overall average = (223+200)/5 = 423/5 = 84.6

OR use deviation from 80:
Deviations: −30, −25, −20, −22, +120 → Sum = +23
Average = 80 + 23/5 = 80 + 4.6 = 84.6 ✓
```

### 🚀 TRICK 15 — The Unit Trap

```
Bar graph title says: "Production (in lakh units)"
Question asks: "Find the total production in crore units."

Read all bars in lakh → add → then convert.
1 crore = 100 lakh
So: divide the lakh answer by 100 to get crores.
```

## 13. COMMON MISTAKE ALERTS

---

### ⚠️ MISTAKE 1 — Using the New Value as Base in % Change

```
Wrong: % Increase from 200 to 250 = (50/250) × 100 = 20%
Right: % Increase from 200 to 250 = (50/200) × 100 = 25%
→ BASE IS ALWAYS THE OLD (STARTING) VALUE. Always.
```

---

### ⚠️ MISTAKE 2 — Confusing "% of" with "% more than"

```
A = 180, B = 120
"A is what % of B?" → (180/120)×100 = 150%
"A is what % more than B?" → [(180−120)/120]×100 = 50%
150% ≠ 50%. These are completely different questions.
```

---

### ⚠️ MISTAKE 3 — Not Checking if Y-axis Starts at Zero

```
If Y-axis starts at 50 (not 0):
A bar that "looks double" another is NOT double.
Always verify: does the scale start at 0? If not, actual values are (scale + bar height).
```

---

### ⚠️ MISTAKE 4 — Adding Stacked Segments Instead of Reading Total

```
Wrong approach: Add each segment value individually = 60 + 40 = 100
Right approach: Read the total height of the bar directly = 100
→ Both give the same answer, but reading the bar directly is faster and less error-prone.
The only time you add segments is when calculating a derived total.
```

---

### ⚠️ MISTAKE 5 — Confusing "A more than B" with "B less than A"

```
A = 250, B = 200
"A is what % more than B?" = (50/200)×100 = 25%
"B is what % less than A?" = (50/250)×100 = 20%
The answers are different because the bases are different!
"More than B" → base = B
"Less than A" → base = A
```

---

### ⚠️ MISTAKE 6 — Applying % Change on Original Value Twice (Chain % Error)

```
Wrong: Production = 100. +20% in Year 1 and +20% in Year 2.
       Year 1 = 100 + 20 = 120. Year 2 = 100 + 20 = 120. Final = 120. ✗
Right: Year 1 = 100 × 1.2 = 120. Year 2 = 120 × 1.2 = 144. ✓
→ Each % increase is on the NEW (updated) value, not the original.
```

---

### ⚠️ MISTAKE 7 — Ignoring Units in the Title

```
Title: "Production in lakh tonnes"
Bar reading: 3.5 → actual value = 3.5 lakh tonnes (not 3.5 tonnes!)
If question asks "in crore tonnes" → 3.5 lakh = 0.035 crore tonnes.
Read unit → extract value → convert if needed → then calculate.
```

---

### ⚠️ MISTAKE 8 — Misreading the Legend in Double Bar Graphs

```
In a double bar graph, if dark = Sales and light = Expenditure:
"What is the total Sales for all years?" → sum only the DARK bars.
"What is the ratio of Sales to Expenditure in 2021?" → dark bar : light bar for 2021.

Always re-confirm the legend for each question if you're unsure.
```

## 🏆 TOP 15 EXAM TIPS

1. **Read the graph fully before touching any question** — title, scale, legend, and trend. This 30-second investment saves you from errors on every subsequent question.

2. **The Y-axis starting point changes everything** — if it doesn't start at zero, every bar value is (baseline + apparent height). Check this first, always.

3. **The base in % change is ALWAYS the old (starting) value** — no matter how the question is phrased. Never use the new value as the base.

4. **"% more than B" and "% less than A" have different bases** — "more than B" uses B as base; "less than A" uses A as base. These give different numerical answers for the same pair of values.

5. **"% of" is not the same as "% more than"** — (A/B)×100 gives percentage of; [(A−B)/B]×100 gives percentage more. Never mix these formulas.

6. **In double bar graphs, confirm which bar represents which variable before reading** — legend misinterpretation leads to systematically wrong answers across the entire set.

7. **For maximum % change, compare fractions — don't compute** — use cross-multiplication on (Rise/Previous bar) fractions. No division needed.

8. **+x% followed by −x% gives a net DECREASE, not zero** — Net change = −(x²/100)%. For +20% then −20%: net = −4%. This is a guaranteed exam trap.

9. **In stacked bar graphs, read segment values as differences between levels** — Segment = (level where it ends) − (level where it starts). Always.

10. **Use the multiplier method for projections** — +20% → multiply by 1.20. Chained changes: multiply all multipliers together. Never apply each % on the original value.

11. **The deviation method is 2x faster for averages** — especially useful when 4–5 values are close together. Pick a midpoint reference, compute small deviations, average the deviations and add to reference.

12. **Approximate aggressively when options are spread out** — if options are 28%, 35%, 42%, 50%, rounding to the nearest 10 and quick division is sufficient. Calculate exactly only when options are within 2–3 units of each other.

13. **Derived values (profit, savings, trade balance) must be recalculated for EACH year** — never assume the year with max sales also has max profit. Expenditure patterns can completely change the profit ranking.

14. **The unit stated in the title applies to all bars in that graph** — read values in title units, calculate in title units, then convert to the unit the question asks for at the very end.

15. **In any DI set, spend the first 45 seconds on pure graph reading, zero calculation** — students who rush to calculate before understanding the graph lose marks on questions they should have gotten right easily.
