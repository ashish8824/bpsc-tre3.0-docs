## 1. WHAT ARE MIXED / COMBINATION CHARTS?

**Mixed Charts** (also called **Combination DI** or **Caselet DI**) are question sets where data is presented using **two or more different chart types simultaneously**.

Instead of a single bar graph or a single pie chart, you are given:
- A **Table + Bar Graph**, or
- A **Pie Chart + Line Graph**, or
- A **Bar Graph + Pie Chart**, or
- A **Table + Pie Chart + Bar Graph** (triple combination), or
- A **Paragraph/Caselet + one or more Charts**

Each chart/source provides **a different piece of data**, and answering any question requires **extracting values from multiple sources and combining them**.

---

### Why Is This The Hardest DI Format?

```
Mixed charts are harder than single-chart DI for three reasons:

REASON 1 — Data is split across sources.
No single chart gives you everything.
You must combine data from Chart 1 AND Chart 2 to answer one question.

REASON 2 — The Linking Variable.
The two charts are connected through one shared variable (e.g., "year",
"department", "city", "product"). Finding and using this link correctly
is the core skill.

REASON 3 — More steps per question.
Single-chart DI: Read value → Apply formula → Answer.
Mixed-chart DI: Read from Chart 1 → Apply intermediate step →
                Read from Chart 2 → Combine → Apply formula → Answer.
Each extra step is a new opportunity for error.
```

---

### Why It Is Heavily Tested

```
Mixed chart DI appears in virtually every major competitive exam:
→ IBPS PO / SBI PO: 15–20 questions per DI set
→ CAT: Complex caselet DI
→ SSC CGL: 2–3 mixed DI sets per paper
→ UPSC CSAT: Paragraph + chart combinations
→ RBI Grade B: Data-rich combination sets

Reason: It tests higher-order thinking — not just reading a chart,
but synthesizing information across multiple sources simultaneously.
```

## 2. TYPES OF MIXED CHART QUESTIONS

---

### Type 1 — Two-Step Calculation (Most Common)

```
Data from Chart 1 feeds into a calculation, whose result is then used in Chart 2.

Example:
Chart 1 (Table): Percentage distribution of employees by department.
Chart 2 (Bar): Total employees per year.

Question: "How many IT employees were there in 2021?"
Step 1 (Chart 2): Read total employees in 2021 = 5000.
Step 2 (Chart 1): IT department = 25% of 5000 = 1250.

Neither chart alone gives the answer. Both are needed.
```

---

### Type 2 — Parallel Reading (Compare Across Charts)

```
Read one value from Chart 1 and one value from Chart 2 for the SAME
entity, then compare or combine.

Example:
Chart 1 (Pie): Revenue distribution by product.
Chart 2 (Line): Total revenue by year.

Question: "What was Product A's revenue in 2020?"
Step 1 (Chart 2/Line): Total revenue in 2020 = ₹8000 Cr.
Step 2 (Chart 1/Pie): Product A = 30% of 8000 = ₹2400 Cr.
```

---

### Type 3 — Verification / Cross-Check

```
Data from both charts must be consistent. One chart provides a total,
the other provides parts. A question may ask if the parts add up to the total.

Example:
Table gives individual values. Bar gives row totals.
"Verify if the row total in the bar matches the sum of individual values in the table."
→ Sum cells from table → compare to bar's height for that year.
```

---

### Type 4 — Growth Rate Application

```
One chart gives a base value. Another gives the growth rate or % change.
You must apply the growth rate to the base value to find a new value.

Example:
Table: Production in 2018 = 400 units.
Line graph: % growth rate in 2019 = 25%.
Question: "What was production in 2019?"
→ 400 × 1.25 = 500 units.
```

---

### Type 5 — Ratio/Proportion Across Charts

```
One chart gives absolute values. Another gives ratios or proportions.

Example:
Bar graph: Total males = 3000, Total females = 2000 in a city.
Pie chart: Employed males = 60% of total males.
Question: "How many males are employed?"
→ 60% of 3000 = 1800.
```

---

### Type 6 — Caselet (Paragraph) + Chart

```
A paragraph provides some data in text. A chart provides additional data.
Questions require integrating text data and chart data.

Example paragraph:
"A company has 3 divisions: A, B, C. Division A has 40% of all employees.
The ratio of male to female in Division A is 3:2."

Chart (bar): Total employees = 2500.

Question: "How many female employees are in Division A?"
Step 1 (paragraph): A has 40% of 2500 = 1000 employees.
Step 2 (paragraph): Male:Female = 3:2 → Female = (2/5) × 1000 = 400.
```

## 4. CONCEPT 1 — THE MASTER FRAMEWORK FOR MIXED CHARTS

> Every mixed chart question follows a predictable pattern. Once you recognize the pattern, you never get confused about which chart to use or in what order.

---

### 4.1 The Five Reading Layers for EACH Chart

**Before reading any question, apply all five reading layers to EACH chart separately.**

```
For CHART 1 (e.g., Table):
Layer 1 — Title: What does this table represent?
Layer 2 — Column headers: What does each column measure?
Layer 3 — Row headers: What does each row represent?
Layer 4 — Units: What unit applies? (₹ Lakhs? %)
Layer 5 — Totals: Any given totals? Any missing cells?

For CHART 2 (e.g., Pie Chart):
Layer 1 — Title: What does this pie chart represent?
Layer 2 — Sector labels: What are the categories?
Layer 3 — Sector values: % or degrees given?
Layer 4 — Total: Is the total given? (CRITICAL for actual values)
Layer 5 — Verify: All sectors sum to 100%?

For CHART 3 (if present): Repeat the same five layers.

TIME INVESTED: 60–90 seconds before Question 1.
TIME SAVED: Avoids re-reading charts for every question.
```

---

### 4.2 Identifying the Linking Variable

**The linking variable is the variable that appears in BOTH charts. It is the bridge between them.**

```
HOW TO IDENTIFY THE LINKING VARIABLE:

Step 1: Look at the X-axis (or row headers) of Chart 1.
Step 2: Look at the X-axis (or row headers) of Chart 2.
Step 3: The variable that is COMMON to both = Linking Variable.

Common linking variables in exams:
→ YEAR: Both charts share time axis (years 2017–2022).
→ DEPARTMENT: One chart has dept breakdown, another has dept totals.
→ CITY/REGION: One chart has city values, another has city totals.
→ PRODUCT: One chart has product %, another has overall totals.
→ COMPANY: Both charts show data for the same set of companies.

Example:
Chart 1 (Pie): Department-wise % of employees for the year 2022.
Chart 2 (Bar): Total employees by year (2017–2022).
Linking variable = YEAR (specifically, 2022 links the two charts).

To find IT employees in 2022:
→ From Bar (Chart 2): Total employees in 2022 = 8000.
→ From Pie (Chart 1): IT = 25% → IT in 2022 = 25% × 8000 = 2000.
```

---

### 4.3 Data Flow — Which Chart Feeds Which Calculation

**Every mixed chart question has a direction of data flow. Identifying this direction tells you which chart to read first.**

```
PATTERN 1 — TOTAL → BREAKDOWN:
Chart A gives the TOTAL. Chart B gives the BREAKDOWN (% or ratio).
Flow: Read total from A → Read % from B → Compute actual value.

"What is the IT department's headcount in 2021?"
Chart A (Bar): Total in 2021 = 5000.          ← Read this FIRST
Chart B (Pie): IT = 30%.                       ← Read this SECOND
Answer: 30% × 5000 = 1500.

PATTERN 2 — BREAKDOWN → TOTAL:
Chart A gives the BREAKDOWN (individual values).
Chart B gives the TOTAL.
Flow: Read individual values from A → Sum → Verify against B.
OR: Read total from B → Use individual values from A to find a share.

PATTERN 3 — BASE → RATE → PROJECTION:
Chart A gives the BASE VALUE.
Chart B gives the GROWTH RATE.
Flow: Read base from A → Read rate from B → Apply multiplier.

"What was production in 2020?"
Chart A (Table): Production in 2019 = 400 units.  ← Base
Chart B (Line): Growth rate 2019→2020 = 15%.      ← Rate
Answer: 400 × 1.15 = 460 units.

PATTERN 4 — CATEGORY → SUB-CATEGORY:
Chart A gives values at the CATEGORY level.
Chart B gives values at the SUB-CATEGORY level within a category.
Flow: Read category total from A → Read sub-category % from B → Compute.

"How many female IT employees are there?"
Chart A (Bar): IT department = 1200 employees.  ← Category total
Chart B (Table): Female in IT = 40%.            ← Sub-category %
Answer: 40% × 1200 = 480.
```

## 5. CONCEPT 2 — TABLE + BAR GRAPH COMBINATION

---

### What Each Chart Provides

```
TABLE: Usually provides the BREAKDOWN or DETAILED data.
       (Individual values per category, per year, per sub-group)

BAR GRAPH: Usually provides the TOTAL or AGGREGATE data.
           (Overall totals for each year or category)

Together: Table gives the "parts"; Bar gives the "whole."
```

---

### Complete Example — Table + Bar Graph

```
BAR GRAPH: Total students enrolled in a college (in hundreds)
  2018: 20 | 2019: 25 | 2020: 22 | 2021: 30 | 2022: 35

TABLE: % distribution of students by stream (row totals = 100%)
| Year | Science | Commerce | Arts | Voc. |
|------|---------|----------|------|------|
| 2018 |   40%   |   30%    | 20%  | 10%  |
| 2019 |   35%   |   32%    | 22%  | 11%  |
| 2020 |   38%   |   28%    | 24%  | 10%  |
| 2021 |   42%   |   30%    | 18%  |  10% |
| 2022 |   45%   |   28%    | 17%  | 10%  |

(Bar is in hundreds. Convert: 2018 = 2000 students, 2019 = 2500, etc.)

LINKING VARIABLE: Year (both bar and table share years 2018–2022).

QUESTION 1: "How many Science students were enrolled in 2021?"
Step 1 (Bar): Total in 2021 = 30 × 100 = 3000 students.
Step 2 (Table): Science in 2021 = 42%.
Answer: 42% × 3000 = 1260 students.

QUESTION 2: "By what % did Science students increase from 2019 to 2022?"
2019 Science: 35% × 2500 = 875.
2022 Science: 45% × 3500 = 1575.
% change = [(1575−875)/875] × 100 = (700/875) × 100 = 80%.

QUESTION 3: "In 2020, Commerce students are what % more than Arts students?"
2020 total = 2200.
Commerce = 28% × 2200 = 616. Arts = 24% × 2200 = 528.
Same year (same total) → use % directly for this question!
% more = [(28−24)/24] × 100 = (4/24) × 100 = 16.67%.

QUESTION 4: "What is the total number of Vocational students across all years?"
2018: 10%×2000=200, 2019: 11%×2500=275, 2020: 10%×2200=220,
2021: 10%×3000=300, 2022: 10%×3500=350.
Total = 200+275+220+300+350 = 1345.

KEY INSIGHT: In Table+Bar combinations, NEVER add the percentages from
the table across years. Each year's % must be multiplied by THAT year's total.
10%+10%+10%+10%+10% ≠ total Vocational. Must compute per year.
```

## 6. CONCEPT 3 — TABLE + PIE CHART COMBINATION

---

### What Each Chart Provides

```
PIE CHART: Gives the PERCENTAGE DISTRIBUTION at one level (overall or a year).
TABLE: Gives DETAILED breakdown within each category (sub-categories,
       gender split, region split, etc.)

Together: Pie gives the top-level split; Table gives the next-level breakdown.
```

---

### Complete Example — Table + Pie Chart

```
PIE CHART: Distribution of 6000 employees by Department (Year 2022)
  Sales: 30% | IT: 25% | HR: 15% | Finance: 20% | Ops: 10%

TABLE: Gender breakdown within each department
| Department | Male % | Female % |
|------------|--------|----------|
| Sales      |   60%  |    40%   |
| IT         |   70%  |    30%   |
| HR         |   35%  |    65%   |
| Finance    |   55%  |    45%   |
| Ops        |   80%  |    20%   |

LINKING VARIABLE: Department (both charts share the same departments).

THE TWO-STEP PROTOCOL:
Step 1 (Pie): Find the department's total = Department% × Grand Total.
Step 2 (Table): Find the sub-group value = Sub-group% × Department total.

QUESTION 1: "How many female IT employees are there?"
Step 1 (Pie): IT = 25% × 6000 = 1500.
Step 2 (Table): Female in IT = 30%.
Answer: 30% × 1500 = 450.

QUESTION 2: "How many male Sales employees are there?"
Step 1: Sales = 30% × 6000 = 1800.
Step 2: Male in Sales = 60%.
Answer: 60% × 1800 = 1080.

QUESTION 3: "Total female employees across all departments?"
Female per department:
Sales:   40% × 1800 = 720
IT:      30% × 1500 = 450
HR:      65% × 900  = 585
Finance: 45% × 1200 = 540
Ops:     20% × 600  = 120
Total females = 720+450+585+540+120 = 2415.

QUESTION 4: "Female HR employees are what % of total female employees?"
Female HR = 585. Total females = 2415.
% = (585/2415) × 100 = 24.22%.

CRITICAL TRAP in Table+Pie combinations:
WRONG: "Female IT employees = 30% of 6000 = 1800." (Applying table % to grand total)
RIGHT: First get IT count (1500), then apply female % (30%) → 450.
The table % must ALWAYS be applied to the department count, not the grand total.
```

## 7. CONCEPT 4 — TABLE + LINE GRAPH COMBINATION

---

### What Each Chart Provides

```
TABLE: Provides ABSOLUTE VALUES for each entity across time periods.
LINE GRAPH: Provides GROWTH RATES, INDEX values, or TREND DATA.

Together: Table gives the starting point; Line gives the trajectory.
```

---

### Complete Example — Table + Line Graph

```
TABLE: Base production of factories (in thousand units) in 2018
| Factory | A   | B   | C   | D   |
|---------|-----|-----|-----|-----|
| 2018    | 200 | 150 | 180 | 120 |

LINE GRAPH: Year-on-year growth rate (%) for each factory
| Year      | 2019 | 2020 | 2021 | 2022 |
|-----------|------|------|------|------|
| Factory A |  20% |  15% |  25% |  10% |
| Factory B |  25% |  20% |  10% |  15% |
| Factory C |  10% |  30% |  20% |  15% |
| Factory D |  15% |  10% |  25% |  20% |

QUESTION 1: "What was Factory A's production in 2020?"
Step 1 (Table): A_2018 = 200.
Step 2 (Line): Growth 2018→2019 = 20%. A_2019 = 200×1.20 = 240.
Step 3 (Line): Growth 2019→2020 = 15%. A_2020 = 240×1.15 = 276 thousand units.

QUESTION 2: "Which factory had the highest production in 2019?"
A_2019 = 200×1.20 = 240.
B_2019 = 150×1.25 = 187.5.
C_2019 = 180×1.10 = 198.
D_2019 = 120×1.15 = 138.
Highest = Factory A (240 thousand units).

QUESTION 3: "By what % did Factory C's production grow from 2018 to 2021?"
C_2018 = 180.
C_2019 = 180×1.10 = 198.
C_2020 = 198×1.30 = 257.4.
C_2021 = 257.4×1.20 = 308.88.
% change from 2018 to 2021 = [(308.88−180)/180]×100 = (128.88/180)×100 = 71.6%.

IMPORTANT: In chained growth, ALWAYS apply each year's rate to the UPDATED value.
WRONG: 200 + (20%+15%+25%+10%)×200 = 200×1.70 = 340 (WRONG!).
RIGHT: 200×1.20×1.15×1.25×1.10 = 200×1.9041 = 380.82 (correct compound growth).
```

## 8. CONCEPT 5 — BAR GRAPH + PIE CHART COMBINATION

---

### What Each Chart Provides

```
BAR GRAPH: Usually provides ABSOLUTE values over time (multiple years/categories).
PIE CHART: Usually provides PERCENTAGE DISTRIBUTION for ONE specific year or scenario.

Together: Bar shows trend; Pie shows composition at a specific point.
```

---

### Complete Example — Bar Graph + Pie Chart

```
BAR GRAPH: Annual Revenue of a Company (in ₹ Crores)
  2018: 400 | 2019: 500 | 2020: 450 | 2021: 600 | 2022: 750

PIE CHART: Revenue distribution by product (for Year 2022 only)
  Product A: 35% | Product B: 25% | Product C: 20% | Product D: 20%

LINKING VARIABLE: 2022 (specific year shared between bar and pie).

QUESTION 1: "What is the revenue from Product A in 2022?"
Step 1 (Bar): Total revenue in 2022 = ₹750 Cr.
Step 2 (Pie): Product A = 35%.
Answer: 35% × 750 = ₹262.5 Cr.

QUESTION 2: "By what % did total revenue increase from 2020 to 2022?"
Both from Bar graph:
Old = 2020 = 450. New = 2022 = 750.
% change = [(750−450)/450] × 100 = (300/450) × 100 = 66.67%.
(Only bar graph needed here — pie chart not involved.)

QUESTION 3: "Product C's 2022 revenue is what % more than Product D's 2022 revenue?"
Both from 2022 (same total) → use pie % directly.
C = 20%, D = 20%. They are equal → 0% more.

QUESTION 4: "If revenue in 2023 grows by 20% over 2022 and Product B maintains
the same 25% share, what will Product B's revenue be in 2023?"
Step 1 (Bar + growth): 2023 revenue = 750 × 1.20 = ₹900 Cr.
Step 2 (Pie + assumption): Product B = 25% × 900 = ₹225 Cr.

IMPORTANT TRAP in Bar+Pie combinations:
The pie chart shows ONLY the year it's labelled for (e.g., 2022).
DO NOT apply pie percentages to other years' bar values unless explicitly told
that the distribution is the same across years.
```

## 9. CONCEPT 6 — BAR GRAPH + LINE GRAPH COMBINATION

---

### What Each Chart Provides

```
BAR GRAPH: Absolute values (revenue, production, headcount) over time.
LINE GRAPH: Rates, ratios, percentages, or a second variable over time.

These two are often plotted on DUAL AXES (left axis for bars, right axis for line).
This is the most visually complex combination.
```

---

### Complete Example — Bar + Line (Dual Axis)

```
BAR GRAPH (Left Y-axis, ₹ Crores): Revenue
LINE GRAPH (Right Y-axis, %): Profit Margin (= Profit/Revenue × 100)

| Year | Revenue (Bar) | Profit Margin % (Line) |
|------|--------------|------------------------|
| 2018 |     400      |          10%           |
| 2019 |     500      |          12%           |
| 2020 |     450      |           8%           |
| 2021 |     600      |          15%           |
| 2022 |     750      |          18%           |

Derived values (must compute):
Profit = Revenue × (Margin %/100)
2018: 400×0.10 = 40 | 2019: 500×0.12 = 60 | 2020: 450×0.08 = 36
2021: 600×0.15 = 90 | 2022: 750×0.18 = 135

QUESTION 1: "What was the profit in 2021?"
Step 1 (Bar): Revenue 2021 = 600.
Step 2 (Line): Profit margin 2021 = 15%.
Answer: 600 × 0.15 = ₹90 Crores.

QUESTION 2: "In which year was absolute profit maximum?"
Compute profit for each year (as above). Maximum = 135 → 2022.

QUESTION 3: "Revenue in 2022 is what % more than Revenue in 2018?"
Both from bar: [(750−400)/400]×100 = (350/400)×100 = 87.5%.

QUESTION 4: "By what % did profit change from 2020 to 2021?"
Profit 2020 = 36. Profit 2021 = 90.
% change = [(90−36)/36]×100 = (54/36)×100 = 150%.

DUAL AXIS CRITICAL RULES:
→ Bar values → LEFT axis.
→ Line values → RIGHT axis.
→ NEVER mix: don't read the bar's value from the right axis.
→ NEVER compare bar and line values directly (different units/scales).
→ Only use them together when a formula EXPLICITLY connects them (e.g., Profit = Revenue × Margin).
```

## 10. CONCEPT 7 — PIE CHART + LINE GRAPH COMBINATION

---

### What Each Chart Provides

```
PIE CHART: Static percentage distribution (at one point in time, or overall).
LINE GRAPH: Dynamic trend (how a variable changed over multiple time periods).

Together: Pie gives the composition at one moment;
          Line shows how totals or rates evolved over time.
```

---

### Complete Example — Pie + Line

```
PIE CHART: Distribution of total exports by product type (for the entire period 2018–2022)
  Electronics: 40% | Textiles: 25% | Chemicals: 20% | Others: 15%

LINE GRAPH: Total annual exports (in ₹ Billion)
  2018: 100 | 2019: 120 | 2020: 110 | 2021: 140 | 2022: 160

LINKING VARIABLE: The pie applies to the CUMULATIVE total (or each year's total
if the pie represents the same distribution each year — READ THE TITLE CAREFULLY).

CASE 1: Pie represents OVERALL distribution (cumulative total):
Total exports 2018–2022 = 100+120+110+140+160 = 630 billion.
Electronics total = 40% × 630 = 252 billion.

CASE 2: Pie represents distribution in ONE specific year (say 2022):
Electronics in 2022 = 40% × 160 = 64 billion.

ALWAYS READ THE PIE CHART TITLE to confirm which case applies.

QUESTION: "By what % did total exports grow from 2019 to 2022?"
Only line graph needed: [(160−120)/120]×100 = (40/120)×100 = 33.33%.

QUESTION: "What were Electronics exports in 2021?" (assuming same % each year)
Step 1 (Line): Total 2021 = 140.
Step 2 (Pie): Electronics = 40%.
Answer: 40% × 140 = 56 billion.
```

## 11. CONCEPT 8 — TRIPLE COMBINATION CHARTS

---

### How to Handle Three Charts

```
Triple combination charts appear in IBPS PO and SBI PO Mains papers.
One DI set uses THREE different chart types, each providing a different data layer.

STRATEGY FOR TRIPLE CHARTS:

Step 1: Read all three charts using the Five Reading Layers.
        Identify title, axis, unit, and scale for each.

Step 2: Identify the linking variable for EACH pair of charts.
        Chart 1 ↔ Chart 2: linked by [X]
        Chart 2 ↔ Chart 3: linked by [Y]
        Chart 1 ↔ Chart 3: linked by [Z]

Step 3: For each question, identify which charts are needed and in what order.

Step 4: Follow the data flow: which chart provides the "input" and which provides the "modifier."
```

---

### Complete Example — Table + Bar + Pie

```
TABLE: % of employees by department (rows = departments, columns = years)
| Dept    | 2020 | 2021 | 2022 |
|---------|------|------|------|
| Sales   |  28% |  30% |  32% |
| IT      |  25% |  27% |  25% |
| HR      |  15% |  12% |  13% |
| Finance |  20% |  18% |  17% |
| Ops     |  12% |  13% |  13% |

BAR GRAPH: Total employees by year
  2020: 4000 | 2021: 5000 | 2022: 6000

PIE CHART: Gender split for the IT department (for 2022)
  Male: 65% | Female: 35%

LINKING VARIABLES:
→ Table + Bar: linked by YEAR (2020, 2021, 2022).
→ Bar + Pie: linked by "IT department in 2022."
→ Table + Pie: indirectly via IT department.

QUESTION 1: "How many IT employees were there in 2021?"
Step 1 (Bar): Total 2021 = 5000.
Step 2 (Table): IT in 2021 = 27%.
Answer: 27% × 5000 = 1350.

QUESTION 2: "How many female IT employees are there in 2022?"
Step 1 (Bar): Total 2022 = 6000.
Step 2 (Table): IT in 2022 = 25% → IT count = 25% × 6000 = 1500.
Step 3 (Pie): Female in IT = 35% → Female IT = 35% × 1500 = 525.

QUESTION 3: "By what % did Sales department headcount change from 2020 to 2022?"
2020 Sales = 28% × 4000 = 1120.
2022 Sales = 32% × 6000 = 1920.
% change = [(1920−1120)/1120]×100 = (800/1120)×100 = 71.43%.

QUESTION 4: "Total IT employees across 2020, 2021, and 2022?"
2020: 25%×4000 = 1000.
2021: 27%×5000 = 1350.
2022: 25%×6000 = 1500.
Total = 1000+1350+1500 = 3850.

This question uses the table AND the bar graph — the pie is not needed.
Identify which charts are needed BEFORE computing.
```

## 12. CONCEPT 9 — TWO PIE CHARTS (COMPARATIVE)

> This is the most heavily tested mixed-chart format. Two pie charts side by side represent data for two different time periods. The traps here are legendary.

---

### The Golden Rules for Two Pie Charts

```
RULE 1: EACH PIE HAS ITS OWN TOTAL.
The totals are almost always DIFFERENT.
Never use one pie's total for the other pie's calculations.

RULE 2: NEVER COMPARE % ACROSS PIES DIRECTLY.
30% in Pie 1 (total=2000) ≠ 30% in Pie 2 (total=5000) in value.
Always convert to actual values using each pie's own total.

RULE 3: % change between pies uses ACTUAL VALUES, not percentages.
% change = [(Actual value in Pie 2 − Actual value in Pie 1) / Actual value in Pie 1] × 100

RULE 4: A falling % share does NOT mean a falling value.
If total grew fast enough, even a smaller % can give a larger value.
```

---

### Complete Example — Two Pie Charts

```
PIE CHART 1 (Year 2018, Total Expenditure = ₹8000 Crores):
  Education: 25% | Health: 20% | Defence: 30% | Transport: 15% | Others: 10%

PIE CHART 2 (Year 2022, Total Expenditure = ₹12000 Crores):
  Education: 20% | Health: 25% | Defence: 25% | Transport: 20% | Others: 10%

ACTUAL VALUES:

Sector       | 2018 Value      | 2022 Value
Education    | 25%×8000 = 2000 | 20%×12000 = 2400
Health       | 20%×8000 = 1600 | 25%×12000 = 3000
Defence      | 30%×8000 = 2400 | 25%×12000 = 3000
Transport    | 15%×8000 = 1200 | 20%×12000 = 2400
Others       | 10%×8000 = 800  | 10%×12000 = 1200

QUESTION 1: "By what % did Education expenditure change from 2018 to 2022?"
Old = 2000, New = 2400.
% change = [(2400−2000)/2000]×100 = (400/2000)×100 = 20% INCREASE.
(Despite share FALLING from 25% to 20% — actual value ROSE by 20%!)

QUESTION 2: "Which sector had the highest % increase in expenditure?"
Education: (400/2000)×100 = 20%
Health: (1400/1600)×100 = 87.5% ← Maximum
Defence: (600/2400)×100 = 25%
Transport: (1200/1200)×100 = 100%... Wait:
Transport: (2400−1200)/1200×100 = 1200/1200×100 = 100% ← Actually maximum!
Others: (400/800)×100 = 50%

Maximum = Transport (100% increase). ✓

QUESTION 3: "By how many percentage points did Education's share change?"
2018 = 25%, 2022 = 20%.
Change = 25−20 = 5 percentage points (decrease).
(NOT 20% decrease — that's the % change in actual value, a different question.)

QUESTION 4: "In 2022, Health is what % more than Education?"
Both from 2022 (same total = 12000) → can use % directly.
Health=25%, Education=20%. "More than Education" → Base = 20%.
% more = [(25−20)/20]×100 = 25%.
Verify: 3000 vs 2400. [(3000−2400)/2400]×100 = 25% ✓.
```

## 13. CONCEPT 10 — CASELET / PARAGRAPH + CHARTS

---

### What Is a Caselet?

```
A CASELET is a paragraph that provides data in TEXT form.
It replaces (or supplements) one of the visual charts.

Typical caselet content:
→ Ratios: "The ratio of male to female employees is 3:2."
→ Percentages: "40% of all students chose Science."
→ Absolute values: "The company has 1200 employees."
→ Conditions: "Each department contributes equally to the total."
→ Growth rates: "Sales grew by 15% annually."
→ Combinations: "Of the 2000 students, 60% are from urban areas,
  and of those, 45% are female."
```

---

### Strategy for Caselet + Chart

```
STEP 1: Read the caselet CAREFULLY and extract ALL data points.
        Write them down explicitly:
        → Total = X, Ratio A:B = m:n, % of C = p%.

STEP 2: Read the chart with the Five Reading Layers.
        Note what the chart provides that the caselet doesn't.

STEP 3: Identify the Linking Variable.
        What information connects the caselet and the chart?

STEP 4: For each question, decide:
        → Does this question need caselet data only?
        → Does this question need chart data only?
        → Does this question need BOTH (most common)?

STEP 5: Follow the Data Flow:
        Caselet usually provides ratios/percentages → Chart provides total.
        OR: Chart provides breakdown → Caselet provides total or further condition.
```

---

### Complete Example — Caselet + Bar Graph

```
CASELET:
"A company has 5 divisions: A, B, C, D, and E.
Division A employs 30% of the total workforce.
Division B employs 20% of the total workforce.
The ratio of employees in Division C to Division D is 3:2.
Division E has the same number of employees as Division B.
The ratio of male to female employees in Division A is 2:1."

BAR GRAPH: Total workforce by year
  2018: 1000 | 2019: 1200 | 2020: 1100 | 2021: 1500 | 2022: 2000

LINKING VARIABLE: Year (caselet gives % splits; bar gives total by year).

Extracting all data from the caselet:
→ A = 30%, B = 20%, E = 20% (same as B).
→ C + D = 100−30−20−20 = 30%.
→ C:D = 3:2 → C = (3/5)×30% = 18%, D = (2/5)×30% = 12%.
→ Male:Female in A = 2:1.

QUESTION 1: "How many employees are in Division A in 2022?"
Step 1 (Bar): Total 2022 = 2000.
Step 2 (Caselet): A = 30%.
Answer: 30% × 2000 = 600.

QUESTION 2: "How many female employees are in Division A in 2021?"
Step 1 (Bar): Total 2021 = 1500.
Step 2 (Caselet): A = 30% → A_2021 = 450.
Step 3 (Caselet): Male:Female = 2:1 → Female = (1/3)×450 = 150.

QUESTION 3: "How many employees are in Division C in 2019?"
Step 1 (Bar): Total 2019 = 1200.
Step 2 (Caselet): C = 18%.
Answer: 18% × 1200 = 216.

QUESTION 4: "What is the ratio of Division C employees to Division D employees in any year?"
C:D = 18%:12% = 18:12 = 3:2.
(Same total → ratio of % = ratio of values. Constant across all years.)

QUESTION 5: "By what % did Division B's headcount grow from 2018 to 2022?"
B_2018 = 20%×1000 = 200. B_2022 = 20%×2000 = 400.
% change = [(400−200)/200]×100 = 100%.
```

## 14. MASTER TRICKS & SHORTCUTS

---

### 🚀 TRICK 1 — The PARSE Method (Solves Every Mixed Chart)

```
Before touching any question, PARSE the chart set:

P — PURPOSE: What does each chart represent? (Read title of each)
A — AXIS: What are the axes / labels of each chart?
R — RELATIONSHIP: What is the linking variable between charts?
S — SCALE/UNIT: What is the unit in each chart? Are they the same?
E — EXTRACT: Extract all constant/static values that appear everywhere.

PARSE takes 90 seconds. It prevents 90% of errors.

Example for Bar + Pie:
P: Bar = total revenue by year. Pie = product-wise % for 2022.
A: Bar X = years, Y = ₹ Crores. Pie = sectors in %.
R: 2022 is the link (bar provides 2022 total; pie gives 2022 breakdown).
S: Bar in ₹ Crores. Pie in %. Use bar total to convert pie % to values.
E: 2022 total = 750 Cr (read from bar). All pie percentages (35%, 25%, 20%, 20%).
```

---

### 🚀 TRICK 2 — Identify "Total Carrier" and "% Carrier"

```
In any mixed chart, one chart is the TOTAL CARRIER (provides the whole)
and the other is the PERCENT CARRIER (provides the parts as %).

TOTAL CARRIER → Read this FIRST.
PERCENT CARRIER → Read this SECOND. Apply to the total.

Common pattern:
Bar graph = Total Carrier (gives absolute total for each year).
Pie chart = % Carrier (gives distribution as percentages).
Table = % Carrier (gives sub-category breakdowns as %).

Formula: Actual value = (% from Percent Carrier) × (Total from Total Carrier) / 100.

Recognition signal:
→ A chart has values in absolute numbers → likely the Total Carrier.
→ A chart has values in % → definitely the % Carrier.
→ A caselet with "X% of total" → % Carrier (needs the total from chart).
```

---

### 🚀 TRICK 3 — The Linking Variable Checklist

```
When you can't figure out which chart to use first, run this checklist:

Question asks about: [specific year + specific category]
Chart 1 covers: [years] in [a specific category] context
Chart 2 covers: [categories] in [a specific year] context

Match the question's "specific year" to the chart that provides year-wise data.
Match the question's "specific category" to the chart that provides category data.

Both charts are needed ONLY when the question requires both.

TEST: "Can I answer this using only ONE chart?"
If YES → use only that chart. Don't overcomplicate.
If NO → identify the flow: which chart gives input for the other?
```

---

### 🚀 TRICK 4 — Carry Intermediate Results Forward

```
In multi-step mixed chart questions, you compute an intermediate value
from Chart 1 and then use it in Chart 2.

MISTAKE: Forgetting the intermediate result and re-reading Chart 1.
SOLUTION: Write the intermediate result clearly before moving to Chart 2.

Protocol:
Step 1: Read Chart 1. Compute. Write: "Intermediate = [X]."
Step 2: Read Chart 2. Apply [X]. Compute final answer.

Example:
"Female IT employees in 2022."
Intermediate: IT total in 2022 = 1500. [Write this down.]
Final: Female IT = 35% × 1500 = 525.

Never try to hold two partial calculations in your head simultaneously.
Write the intermediate, then proceed.
```

---

### 🚀 TRICK 5 — Two Pie Chart % Change Trap (Most Tested)

```
In comparative (two) pie charts with DIFFERENT totals:

% change in actual value ≠ change in % share.

A SECTOR'S SHARE FELL from 30% to 20% → DOES NOT MEAN VALUE FELL.
If Total 1 = 2000 and Total 2 = 5000:
Value 1 = 30%×2000 = 600. Value 2 = 20%×5000 = 1000. Value ROSE!

The THREE-STEP RULE for two pie chart questions:
Step 1: Compute Actual Value in Pie 1 = %₁ × Total₁.
Step 2: Compute Actual Value in Pie 2 = %₂ × Total₂.
Step 3: Compute % change = [(V₂−V₁)/V₁] × 100.

Never skip Steps 1 and 2. Never compare % across different pies directly.

QUICK CHECK (no calculation):
If %₂ × Total₂ > %₁ × Total₁ → value increased.
This can happen even when %₂ < %₁ if Total₂ >> Total₁.

Break-even test:
%₂/Total₂ = %₁/Total₁ → values are equal (% change = 0).
Simplify: %₂ × Total₁ = %₁ × Total₂ → cross-multiplication check.
```

---

### 🚀 TRICK 6 — Same Year / Same Total Shortcut

```
When two values come from the SAME PIE CHART (same total):
→ Ratio of values = Ratio of percentages. Use % directly.
→ "% more than" = [(A%−B%)/B%]×100. No total needed.

When two values come from DIFFERENT PIE CHARTS (different totals):
→ MUST compute actual values first.
→ Never compare percentages across charts.

Quick test: "Are both values from the same total?"
YES → Use % directly for ratio and % more/less questions.
NO → Compute actual values first.

Saves time on easy ratio questions within the same pie chart.
Prevents errors on cross-chart comparison questions.
```

---

### 🚀 TRICK 7 — Compound Growth Short Cut

```
When growth rates are applied consecutively (table+line graph type):

Instead of computing year by year, multiply all multipliers together:
Final value = Initial × M₁ × M₂ × M₃ × ...
Where Mₙ = (1 + growth rate/100) for each year.

Example:
Initial = 200. Growth rates: 20%, 15%, 25%, 10%.
Final = 200 × 1.20 × 1.15 × 1.25 × 1.10.

Compute step by step (faster in exam):
200 × 1.20 = 240.
240 × 1.15 = 276.
276 × 1.25 = 345.
345 × 1.10 = 379.5.

DO NOT add growth rates: 200 × (1 + 0.20+0.15+0.25+0.10) = 200×1.70 = 340. WRONG.
Each rate applies to the updated value, not the original.
```

---

### 🚀 TRICK 8 — Dual Axis: Left for Bars, Right for Lines

```
In dual-axis (Bar + Line) charts:
→ Read BAR heights using the LEFT Y-axis.
→ Read LINE values using the RIGHT Y-axis.

NEVER cross-read: never use the right axis for bars or left axis for lines.

Identification:
→ Bar = absolute values (Revenue ₹ Cr, Production units). Usually on left.
→ Line = rates / percentages / secondary variable. Usually on right.

EXAM TRAP: Both a bar and a line appear at the same height visually.
           But their actual values are COMPLETELY DIFFERENT because
           they use DIFFERENT scales (left vs right axis).

Example: Bar at 500 (read from left: ₹500 Cr) and
         Line at same visual height (read from right: 50%).
         "Bar = Line" visually means NOTHING about their values.
```

---

### 🚀 TRICK 9 — Caselet Data Extraction Template

```
When given a caselet, extract data into a structured template:

Total = ___
Category breakdown:
  A = ___%, B = ___%, C = ___%, ...
  (Compute missing categories: Missing = 100 − Sum of known %)

Ratios within categories:
  Male:Female in A = ___:___
  X:Y in B = ___:___

Derived quantities:
  Value of A = A% × Total = ___
  Male in A = (male ratio / total ratio) × Value of A = ___

Doing this BEFORE reading questions means any question is just:
"pick the relevant values from my template and apply the formula."

Never read a caselet question by question — extract ALL data first.
```

---

### 🚀 TRICK 10 — Total from Partial Data (Reverse Calculation)

```
When one chart gives a sub-category's absolute value AND the other chart
gives that sub-category's percentage → you can find the total.

Formula: Total = Absolute value × (100 / Percentage)

Example:
Bar graph: IT employees = 600 (absolute value from bar).
Table: IT = 20% of total employees.
Find total employees:
Total = 600 × (100/20) = 600 × 5 = 3000.

This reverse calculation appears in complex mixed chart questions.
Recognize it when you see: one chart has a %, another has the actual value.
```

---

### 🚀 TRICK 11 — Percentage Point vs % Change (Critical in Two-Pie Questions)

```
TWO ENTIRELY DIFFERENT THINGS:

Percentage Point change:
→ Simple arithmetic difference of two percentages.
→ "How many percentage points did Education's share change?"
→ 25% − 20% = 5 percentage points decrease.
→ NO total needed. NO actual values needed.

% change in actual value:
→ Formula: [(New value − Old value) / Old value] × 100.
→ "By what % did Education's expenditure change?"
→ Need actual values: Old = 25%×8000=2000, New = 20%×12000=2400.
→ [(2400−2000)/2000]×100 = 20% increase.

KEY WORDS:
"percentage points" → simple subtraction of %s.
"by what %" / "% change" → compute actual values + formula.
```

---

### 🚀 TRICK 12 — The "Only One Chart Needed" Quick Screen

```
Before computing anything, ask: "Does this question need BOTH charts?"

Questions that need ONLY the Bar/Line (no table/pie):
→ "What is the total revenue in 2022?" (read bar directly)
→ "By what % did total employees grow from 2019 to 2022?" (bar only)
→ "In which year was revenue highest?" (bar only)

Questions that need ONLY the Table/Pie (no bar):
→ "IT is what % more than HR (within same year)?" (same total → pie/table only)
→ "What is the ratio of A to B in the given distribution?" (same pie)
→ "Which product has the highest % share?" (pie only)

Questions that need BOTH:
→ "How many IT employees were there in 2021?" (bar for total + pie for %)
→ "By what % did Product A revenue change from 2019 to 2022?" (bar for both years + pie for %)
→ "Female employees in HR in 2022?" (bar for total → table for HR% → table for female%)

Identifying "only one chart" questions saves significant time.
Don't unnecessarily read both charts when one suffices.
```

---

### 🚀 TRICK 13 — Quick % from Friendly Totals

```
When a bar/line gives a "friendly" total and a pie/table gives a %:

Total = 5000: V = X% → Value = X×50 (multiply % by 50).
Total = 2000: V = X% → Value = X×20.
Total = 4000: V = X% → Value = X×40.
Total = 1000: V = X% → Value = X×10.
Total = 3000: V = X% → Value = X×30.
Total = 6000: V = X% → Value = X×60.
Total = 8000: V = X% → Value = X×80.

Example: Total = 5000, IT = 30%.
IT count = 30 × 50 = 1500. (Faster than 30/100 × 5000.)

Memorize: "For total = N thousand, multiply % by N×10."
5000 → multiply by 50.
3000 → multiply by 30. Etc.
```

---

### 🚀 TRICK 14 — Cross-Verification Using Grand Total

```
In combined charts, you can verify your calculation using the grand total.

Example:
Pie chart: 5 sectors summing to 100%.
Bar graph gives total = 8000.

After computing all sector values:
Sum of all computed sector values should = 8000.

If sum ≠ 8000 → you made an arithmetic error in one sector.
Find it before answering further questions.

This takes 20 seconds and prevents systematic errors.

For Table + Bar: Row totals from table × respective row % sums = bar value.
For Caselet + Bar: Sum of all department values (from caselet%) should = bar total.
```

---

### 🚀 TRICK 15 — Unit Consistency Check (Critical in Mixed Charts)

```
Mixed charts often use DIFFERENT units in different charts.

Common mismatches:
Chart 1 (Bar): Revenue in ₹ Crores.
Chart 2 (Table): Expenses in ₹ Lakhs.

BEFORE computing: Convert all values to the SAME unit.
Or compute in each chart's unit and convert ONLY at the final answer.

Another common mismatch:
Chart 1: Values in thousands (Bar at height 25 = 25,000 units).
Chart 2: Values as raw numbers (Table shows 30%).

Conversion: 25 × 1000 = 25,000 (actual value from bar).
Then: 30% × 25,000 = 7,500.

Always: Read units from EACH chart's title/axis independently.
Never assume both charts use the same unit.
If units differ → convert everything to one common unit before computing.
```

## 16. COMMON MISTAKE ALERTS

---

### ⚠️ MISTAKE 1 — Using the Wrong Chart for a Step

```
Wrong: Reading IT department's % from the bar graph (which shows totals, not %).
Right: IT % always comes from the pie/table. Total always comes from the bar.

Prevention: Label each chart: "Chart 1 = Total Carrier" or "Chart 1 = % Carrier."
            Then: Step 1 always reads the Total Carrier.
                  Step 2 always reads the % Carrier.
```

---

### ⚠️ MISTAKE 2 — Applying Table % to Grand Total (Skipping the First Step)

```
Wrong: "Female IT employees = (Female IT %) × Grand Total."
Right: Female IT = (Female IT %) × (IT department total).
       IT department total = (IT %) × Grand Total.

You MUST apply the table % to the DEPARTMENT total, not the grand total.
The two-step process cannot be shortened. Ever.
```

---

### ⚠️ MISTAKE 3 — Comparing % Across Two Pie Charts

```
Wrong: "Health's share rose from 20% to 25%, so health grew by 5%."
Right: Compute actual values using each pie's own total.
       Health grew by [(actual₂ − actual₁) / actual₁] × 100.

The % change in SHARE is a percentage point change (5 pp), not a % change in value.
```

---

### ⚠️ MISTAKE 4 — Applying Pie % to the Wrong Year's Bar Total

```
Scenario: Bar shows 2018–2022 totals. Pie shows ONLY 2022 distribution.

Wrong: "Product A in 2019 = 35% × 2019 bar value."
Right: The pie chart's 35% for Product A applies ONLY to 2022.
       For 2019, either a separate distribution is given, or the question explicitly says
       "same distribution throughout."

ALWAYS read the pie chart's title to confirm which year(s) it applies to.
```

---

### ⚠️ MISTAKE 5 — Adding % Values Across Years (Same Error as Single Charts)

```
Wrong: "Total Vocational students = (10%+11%+10%+10%+10%) × single total."
Right: Each year's % must be multiplied by THAT YEAR'S total separately.
       Vocational total = (10%×2000) + (11%×2500) + ... etc.

This is because each year has a DIFFERENT total — the % values cannot be pooled.
```

---

### ⚠️ MISTAKE 6 — Compound Growth: Adding Rates Instead of Multiplying

```
Wrong: Final = Initial × (1 + Sum of all growth rates/100).
Right: Final = Initial × (1+r₁/100) × (1+r₂/100) × ...

Adding rates works ONLY for simple (non-compound) interest.
For year-on-year growth, each rate applies to the UPDATED value.
Always multiply multipliers, never add rates.
```

---

### ⚠️ MISTAKE 7 — Dual Axis: Reading Bar from Right Axis

```
Wrong: Reading Revenue bar's height from the right (%) axis.
Right: Revenue bar → left axis. Growth rate line → right axis.

Visual trap: Both might appear at the "same height" on the graph,
but they have completely different values because of different axis scales.
```

---

### ⚠️ MISTAKE 8 — Caselet: Missing Hidden % or Derived Value

```
Caselet says: "A has 30%, B has 20%, E has same as B."
Students often compute C and D incorrectly because they forget E = 20%.
C+D = 100−30−20−20 = 30%.

Always: After reading a caselet, compute ALL missing category values
before starting questions. List them all.
```

## 17. SOLVED EXAMPLES (BASIC TO ADVANCED)

---

### 📘 EXAMPLE SET 1 — TABLE + BAR GRAPH (BASIC)

**Bar Graph: Total Production of all factories combined (in lakh units)**
```
2018: 50 | 2019: 60 | 2020: 55 | 2021: 70 | 2022: 80
```

**Table: % share of each factory in total production**

| Year | Factory A | Factory B | Factory C | Factory D |
|------|-----------|-----------|-----------|-----------|
| 2018 |    30%    |    25%    |    25%    |    20%    |
| 2019 |    32%    |    28%    |    22%    |    18%    |
| 2020 |    28%    |    30%    |    24%    |    18%    |
| 2021 |    35%    |    27%    |    22%    |    16%    |
| 2022 |    40%    |    25%    |    20%    |    15%    |

**Linking variable: Year**

---

**Q1. What was Factory A's production in 2021?**

```
Step 1 (Bar): Total 2021 = 70 lakh units.
Step 2 (Table): Factory A in 2021 = 35%.
Answer: 35% × 70 = 24.5 lakh units.
```

**Q2. By what % did Factory A's actual production increase from 2018 to 2022?**

```
A_2018 = 30% × 50 = 15.
A_2022 = 40% × 80 = 32.
% change = [(32−15)/15] × 100 = (17/15) × 100 = 113.33%.
```

**Q3. Which factory produced more in 2020 — B or C?**

```
B_2020 = 30% × 55 = 16.5 lakh units.
C_2020 = 24% × 55 = 13.2 lakh units.
Factory B produced more.
(Same total → could also compare % directly: 30% > 24% → B wins.)
```

**Q4. What is the total production of Factory D over all 5 years?**


## 🏆 TOP 15 EXAM TIPS

1. **PARSE every chart set before Question 1** — Purpose, Axis, Relationship, Scale, Extract. This 90-second investment is the most important habit in mixed chart DI. It tells you what each chart provides and how they connect, so you never get lost mid-calculation.

2. **Label each chart as Total Carrier or % Carrier** — The Total Carrier gets read first in every two-step calculation. The % Carrier gets applied to the total. This eliminates the confusion of "which chart do I use first?"

3. **Find and commit to the Linking Variable** — The linking variable (year, department, city, product) is the bridge between charts. State it explicitly: "Charts are linked by Year. 2022 is the linking year for these questions." Never let the link be ambiguous.

4. **Two-step protocol for Table/Pie + Bar combinations** — NEVER apply a table/pie % directly to the grand total. Step 1: find category total (from bar). Step 2: apply sub-category % (from table/pie) to that category total. This two-step process cannot be shortened.

5. **In two-pie chart questions: ALWAYS compute actual values before comparing** — falling % share does NOT mean falling actual value. Compute Actual = % × Own Total for EACH pie separately, then compare. Never use percentages across different pies.

6. **Percentage point ≠ % change** — "by how many percentage points" → subtract the % values (no total needed). "By what %" → compute actual values and use the % change formula. These are completely different calculations for questions that sound almost identical.

7. **Dual axis: bars use left axis, lines use right axis. Never mix** — the two axes have completely different scales and units. Visually identical heights mean nothing about actual value. Always identify which axis applies to which chart element before reading any value.

8. **Compound growth: multiply multipliers, never add rates** — applying 20% + 15% as 35% in one step is wrong. The correct method is: Initial × 1.20 × 1.15. Each rate applies to the updated value from the previous step.

9. **For caselet DI: extract ALL data points before reading Question 1** — list every given percentage, ratio, and absolute value in a structured format. Compute all derived quantities (like "missing category = 100% − sum of given"). This template makes every question a simple look-up.

10. **Use the "Only One Chart Needed?" screen** — before computing, ask if the question needs both charts or just one. Questions about totals (year-wise) need only the bar. Questions about ratios/% within one year may need only the pie/table. Avoid unnecessary chart-reading.

11. **Carry intermediate results in writing** — in multi-step questions, write the intermediate value clearly before moving to the next chart. Never hold two partial calculations in your head simultaneously.

12. **Unit check before every cross-chart calculation** — identify the unit of each chart independently. If one chart uses Lakhs and another uses Crores, convert to a common unit BEFORE combining. Converting mid-calculation causes systematic errors across all questions.

13. **"Total of a product across multiple years" in Bar+Pie** — NEVER add percentages across years and multiply by one total. Compute (% × year's total) for EACH year separately, then add. Each year has a different total.

14. **Verify your computation against the grand total** — after computing all sub-values, their sum should equal the total from the Total Carrier chart. If it doesn't, find the error before answering further questions.

15. **In complex three-chart sets, draw a simple flow diagram** — a quick sketch: "Bar → IT total → Table → Female IT %" shows the exact data path. This 10-second sketch prevents mid-calculation confusion and ensures you read charts in the right order.
