## 1. THE BIG PICTURE — WHAT IS LINEAR SEATING?

**Linear Seating Arrangement** questions give you a set of people and a set of clues about where they sit in a **straight line** (row or bench). You must determine the exact position of each person.

> **Golden Rule #1:** ALWAYS draw the row first. Mark positions 1, 2, 3... from left to right. Fill in who goes where.

> **Golden Rule #2:** Never assume. Only use what the clue explicitly tells you. Distinguish between "immediate left" (position −1) and "to the left" (any position to the left).

---

```
N people sit in a ROW.

LEFT END                                          RIGHT END
[Pos 1] [Pos 2] [Pos 3] [Pos 4] [Pos 5] ... [Pos N]

OR labeled as:
[1]     [2]     [3]     [4]     [5]     ... [N]

## 3. CLUE LANGUAGE — READING EVERY CLUE CORRECTLY

> This section is CRITICAL. Many students lose marks because they misread clues.

---

### 3.1 — Position-Based Clues

```
┌────────────────────────────────┬──────────────────────────────────────────┐
│ CLUE                           │ MEANING                                  │
├────────────────────────────────┼──────────────────────────────────────────┤
│ "A sits at the extreme left"   │ A is at Position 1 (leftmost)            │
│ "A sits at the extreme right"  │ A is at Position N (rightmost)           │
│ "A sits at one of the ends"    │ A is at Position 1 OR Position N         │
│ "A is at the center"           │ A is at the middle position              │
│ "A is in the middle"           │ A is at position (N+1)/2 (odd N only)    │
└────────────────────────────────┴──────────────────────────────────────────┘
```

---

### 3.2 — Relative Position Clues (Most Common!)

```
┌────────────────────────────────────┬──────────────────────────────────────┐
│ CLUE                               │ MEANING                              │
├────────────────────────────────────┼──────────────────────────────────────┤
│ "A sits immediately to the left    │ A is EXACTLY at position (B's pos −1)│
│  of B"                             │ A_pos = B_pos − 1                    │
│                                    │ They are ADJACENT, A is on left      │
├────────────────────────────────────┼──────────────────────────────────────┤
│ "A sits immediately to the right   │ A is EXACTLY at position (B's pos +1)│
│  of B"                             │ A_pos = B_pos + 1                    │
│                                    │ They are ADJACENT, A is on right     │
├────────────────────────────────────┼──────────────────────────────────────┤
│ "A sits to the left of B"          │ A is SOMEWHERE to B's left           │
│                                    │ A_pos < B_pos (not necessarily adj.) │
├────────────────────────────────────┼──────────────────────────────────────┤
│ "A sits to the right of B"         │ A is SOMEWHERE to B's right          │
│                                    │ A_pos > B_pos (not necessarily adj.) │
├────────────────────────────────────┼──────────────────────────────────────┤
│ "A and B are adjacent / next to    │ |A_pos − B_pos| = 1                  │
│  each other"                       │ They sit right next to each other    │
├────────────────────────────────────┼──────────────────────────────────────┤
│ "A and B are not adjacent"         │ |A_pos − B_pos| ≥ 2                  │
├────────────────────────────────────┼──────────────────────────────────────┤
│ "Exactly 2 persons sit between     │ |A_pos − B_pos| = 3                  │
│  A and B"                          │ (2 people between = gap of 3)        │
├────────────────────────────────────┼──────────────────────────────────────┤
│ "A is 3rd to the left of B"        │ A_pos = B_pos − 3                    │
│                                    │ (counted as positions, not persons)  │
├────────────────────────────────────┼──────────────────────────────────────┤
│ "A is 3rd from the left end"       │ A_pos = 3                            │
├────────────────────────────────────┼──────────────────────────────────────┤
│ "A is 3rd from the right end"      │ A_pos = N − 2 (where N = total)      │
└────────────────────────────────────┴──────────────────────────────────────┘
```

---

### 3.3 — The Critical "Between" Formula:

```
╔══════════════════════════════════════════════════════════╗
║  If "N persons sit BETWEEN A and B":                     ║
║  Then |A_position − B_position| = N + 1                  ║
║                                                          ║
║  Example: "2 persons sit between A and B"                ║
║  → Gap between positions = 3 (i.e., pos differ by 3)     ║
║                                                          ║
║  Visual: [A] [_] [_] [B]  → positions 1, 2, 3, 4        ║
║           ↑           ↑                                  ║
║          pos1        pos4  → difference = 3 = 2+1 ✓     ║
╚══════════════════════════════════════════════════════════╝
```

---

### 3.4 — Facing Direction Clues (Two Rows):

```
┌──────────────────────────────────────┬────────────────────────────────────┐
│ CLUE                                 │ MEANING                            │
├──────────────────────────────────────┼────────────────────────────────────┤
│ "A faces B"                          │ A and B are in different rows,     │
│                                      │ sitting directly opposite          │
├──────────────────────────────────────┼────────────────────────────────────┤
│ "A and B face the same direction"    │ Both face North or both face South │
├──────────────────────────────────────┼────────────────────────────────────┤
│ "A faces North"                      │ A looks toward North               │
├──────────────────────────────────────┼────────────────────────────────────┤
│ "Immediate left of A" (facing North) │ The person to A's physical left    │
│                                      │ when A faces North = WEST side     │
├──────────────────────────────────────┼────────────────────────────────────┤
│ "Immediate left of A" (facing South) │ The person to A's physical left    │
│                                      │ when A faces South = EAST side     │
│                                      │ ⚠️ This is the OPPOSITE of         │
│                                      │ facing-North convention!           │
└──────────────────────────────────────┴────────────────────────────────────┘
```

---

### 3.5 — Left/Right When Facing South vs North (CRITICAL!)

```
NORTH-FACING ROW (standard in most exams):
    You face North → Your RIGHT = East direction → Higher position numbers
    You face North → Your LEFT  = West direction → Lower position numbers

    ← LEFT (West / lower pos)      RIGHT (East / higher pos) →
    [1]  [2]  [3]  [4]  [5]  [6]
    All persons face North (↑)

SOUTH-FACING ROW:
    You face South → Your RIGHT = West direction → Lower position numbers
    You face South → Your LEFT  = East direction → Higher position numbers

    ← RIGHT (West / lower pos)     LEFT (East / higher pos) →
    [1]  [2]  [3]  [4]  [5]  [6]
    All persons face South (↓)

⚠️ KEY INSIGHT: When facing South, LEFT and RIGHT are SWAPPED
   compared to what we see on the diagram (physical left becomes their right)!
```

## 4. THE SLOT METHOD — YOUR PRIMARY WEAPON

> **The Slot Method** is the universal technique for all linear seating questions.

---

### How the Slot Method Works:

```
STEP 1: Count total persons → N
STEP 2: Draw N boxes (slots):
         [_] [_] [_] [_] [_] [_]
          1   2   3   4   5   6

STEP 3: Label positions clearly:
         LEFT ←────────────────→ RIGHT
         [_] [_] [_] [_] [_] [_]
          1   2   3   4   5   6

STEP 4: Find the most RESTRICTED clue (gives exact position) → Fill it in first.
STEP 5: Use each new placement to deduce others.
STEP 6: Check all clues against your arrangement.
```

---

### Types of Clue Priority (Fill in this order!):

```
PRIORITY 1 (Fill first): Exact position clues
  → "A sits at position 3"
  → "B sits at the extreme left"
  → "C is 2nd from right"

PRIORITY 2 (Fill second): Relative clues that become exact after Priority 1
  → "D sits immediately to the right of A" (once A is placed)
  → "E is 3 places to the left of B" (once B is placed)

PRIORITY 3 (Fill third): Multiple possibility clues
  → "F sits somewhere between G and H"
  → "X does not sit at either end"

PRIORITY 4 (Use to eliminate): Negative clues
  → "P is not adjacent to Q"
  → "R does not sit at position 4"
```

## 5. TYPE 1 — SINGLE ROW (ALL FACE SAME DIRECTION)

> Most basic type. All persons sit in a single line, all facing the same direction (usually North — toward you as you look at the diagram).

### Worked Example 1 — 6 Persons, Straightforward:

```
PROBLEM:
Six persons A, B, C, D, E, F sit in a row.
(i)   B sits at the extreme left.
(ii)  F sits at the extreme right.
(iii) C is immediately to the right of B.
(iv)  D sits between C and E.
(v)   A is not adjacent to F.

Find the arrangement.

SOLUTION:

Step 1: Draw slots for 6 persons.
[_] [_] [_] [_] [_] [_]
 1   2   3   4   5   6

Step 2: Apply Priority 1 clues (exact positions):
Clue (i):  B at position 1 (extreme left)
Clue (ii): F at position 6 (extreme right)

[B] [_] [_] [_] [_] [F]
 1   2   3   4   5   6

Step 3: Apply Priority 2 clues:
Clue (iii): C immediately to right of B → C at position 2.

[B] [C] [_] [_] [_] [F]
 1   2   3   4   5   6

Step 4: Remaining persons: A, D, E at positions 3, 4, 5.
Clue (iv): D sits between C and E → D is between position 2 (C) and E's position.
           D must be at position 3, 4, or some spot with E on one side and C on other.
           Since C is at 2 and D must be between C and E:
           If D = 3: E must be to the right of D (E > 3). So E = 4 or 5.
           If D = 4: E must be at 5 (to the right). Or E = 3 (to the left of D).
           
           "D sits between C and E" means C...D...E or E...D...C pattern.
           C is at 2. So C...D...E means D > 2 and E > D.
           Try D=3, E=4: C(2)...D(3)...E(4) ✓ → A at 5.
           Try D=3, E=5: C(2)...D(3)...E(5) ✓ → A at 4.
           Try D=4, E=5: C(2)...D(4)...E(5) ✓ → A at 3.

Step 5: Apply negative clue:
Clue (v): A is not adjacent to F (position 6). So A ≠ 5.

Eliminate D=3,E=4→A=5: A=5 is adjacent to F=6. ✗
Remaining options:
  Option 1: D=3, E=5, A=4 → Check A(4) adjacent to F(6)? No (|4-6|=2) ✓
  Option 2: D=4, E=5, A=3 → Check A(3) adjacent to F(6)? No (|3-6|=3) ✓

Both seem valid. But wait, check E...D...C pattern too:
E...D...C means D is between E and C: E < D < C or E > D > C.
Since C=2, E > D > C means D=3 and E=4 or E=5... wait that has E>D>C: E>D>2.
This gives same results. Let me check if "between C and E" specifically means C is to the LEFT.

Clue (iv) says "D sits between C and E" — doesn't specify order.
Both options give D between C and E positionally.

Need another clue to decide. No more clues given.
→ BOTH arrangements are valid, meaning the puzzle as stated has two solutions.
(In well-set exam puzzles, there's always exactly one solution.)

For illustration purposes, let's take Option 1:
[B] [C] [D] [A] [E] [F]
 1   2   3   4   5   6

FINAL ARRANGEMENT: B-C-D-A-E-F
```

```
PROBLEM:
Seven persons P, Q, R, S, T, U, V sit in a row facing North.
(i)   P sits at one of the extreme ends.
(ii)  V is 3rd to the right of P.
(iii) Q is to the immediate left of V.
(iv)  Two persons sit between Q and T.
(v)   U sits to the immediate right of T.
(vi)  R does not sit at either end.
(vii) S is not adjacent to P.

Find the complete arrangement.

SOLUTION:

Step 1: Draw 7 slots.
[_] [_] [_] [_] [_] [_] [_]
 1   2   3   4   5   6   7

Step 2: Clue (i): P at position 1 OR 7.
Try P = 1 first.

[P] [_] [_] [_] [_] [_] [_]
 1   2   3   4   5   6   7

Step 3: Clue (ii): V is 3rd to the right of P.
P at 1 → V at 1+3 = 4.

[P] [_] [_] [V] [_] [_] [_]
 1   2   3   4   5   6   7

Step 4: Clue (iii): Q immediately to left of V → Q at 4-1 = 3.

[P] [_] [Q] [V] [_] [_] [_]
 1   2   3   4   5   6   7

Step 5: Clue (iv): Two persons between Q and T.
Q at 3. |Q-T| = 3 (two persons between = gap of 3).
T at 3+3=6 OR T at 3-3=0 (invalid).
So T at 6.

[P] [_] [Q] [V] [_] [T] [_]
 1   2   3   4   5   6   7

Step 6: Clue (v): U immediately to right of T → U at 7.

[P] [_] [Q] [V] [_] [T] [U]
 1   2   3   4   5   6   7

Step 7: Remaining persons: R and S at positions 2 and 5.
Clue (vi): R not at either end → R ≠ 1 and R ≠ 7. Positions 2 and 5 both satisfy this. ✓ (no help)
Clue (vii): S not adjacent to P(pos 1) → S ≠ 2.
So S at 5, R at 2.

[P] [R] [Q] [V] [S] [T] [U]
 1   2   3   4   5   6   7

Step 8: Verify ALL clues:
(i)  P at extreme left ✓
(ii) V is 3rd to right of P: V at 4, P at 1, 4-1=3 ✓
(iii)Q immediately left of V: Q at 3, V at 4 ✓
(iv) Two between Q(3) and T(6): positions 4,5 → 2 persons ✓
(v)  U immediately right of T: T at 6, U at 7 ✓
(vi) R not at ends: R at 2 ✓
(vii)S not adjacent to P: S at 5, P at 1 → |5-1|=4 ✓

FINAL ARRANGEMENT (left to right):
P — R — Q — V — S — T — U
1    2    3    4    5    6    7

Now try P=7 to check:
V at 7+3=10 (invalid — only 7 positions). So P=7 gives no valid solution.
P=1 confirmed.
```

## 6. TYPE 2 — SINGLE ROW (FACE NORTH / SOUTH SPECIFIED)

> When the facing direction is specified, LEFT and RIGHT clues must be interpreted carefully.

---

### North-Facing Row (Standard):

```
When persons face NORTH:
→ Their RIGHT is to the EAST → higher position numbers in diagram
→ Their LEFT is to the WEST → lower position numbers in diagram
→ LEFT/RIGHT on the diagram = LEFT/RIGHT of the persons ✓ (matches directly)

DIAGRAM:
← WEST (their left)          EAST (their right) →
[1]  [2]  [3]  [4]  [5]  [6]
      ↑    ↑    ↑    ↑
   All facing North (toward you as you read)
```

---

### South-Facing Row (Reversed!):

```
When persons face SOUTH:
→ Their RIGHT is to the WEST → lower position numbers in diagram
→ Their LEFT is to the EAST → higher position numbers in diagram
→ LEFT/RIGHT of persons is OPPOSITE to what you see on paper!

DIAGRAM:
← EAST (their left)          WEST (their right) →
[1]  [2]  [3]  [4]  [5]  [6]
      ↓    ↓    ↓    ↓
   All facing South (away from you as you read)

⚠️ So when the clue says "B is to the immediate LEFT of A" and they face South:
B is at a HIGHER position number than A in the diagram!
```

---

### Worked Example 4 — South-Facing Row:

```
PROBLEM:
Six persons A, B, C, D, E, F sit in a row facing South.
(i)  A sits at position 3 (from west end).

## 12. MASTER TRICKS & SHORTCUTS

---

### 🚀 TRICK 1 — Start with the Most Anchored Clue

```
The MOST ANCHORED clue gives an EXACT position directly.
Look for:
→ "sits at extreme left/right" → exact position 1 or N
→ "3rd from left" → exact position 3
→ "sits at the center" → exact middle position

Place these FIRST before anything else.
```

---

### 🚀 TRICK 2 — The "Between N Persons" Formula

```
"X persons sit between A and B"
→ |A_pos − B_pos| = X + 1

"A and B are adjacent" (0 between)
→ |A_pos − B_pos| = 1

"Exactly 1 person between" → gap = 2
"Exactly 2 persons between" → gap = 3
"Exactly 3 persons between" → gap = 4

Quick pattern: persons between + 1 = gap between positions
```

---

### 🚀 TRICK 3 — Chain Elimination

```
Once you place Person A, immediately check ALL other clues
that reference A to see what you can now determine.

Place A → check all "A is adjacent to...", "X is 2nd to right of A", etc.
Place B (from A's chain) → check all clues referencing B.
Continue until stuck, then use negative/elimination clues.
```

---

### 🚀 TRICK 4 — Try Both Ends for "Either End" Clues

```
When a clue says "P sits at one of the extreme ends":
Try P at the LEFT end first.
  → Build the arrangement as far as possible.
  → If contradiction found → try P at RIGHT end.
  → If P at right also fails → puzzle error.

Usually one case works and the other gives contradiction.
```

---

### 🚀 TRICK 5 — The "Not Adjacent" Shortcut

```
"A is not adjacent to B" is a LIMITING clue, not a PLACING clue.
Use it LAST (after placing others by positive clues).
Then check if the remaining positions for A and B are non-adjacent.
```

---

### 🚀 TRICK 6 — Gender Alternating Pattern

```
If: No two males adjacent AND no two females adjacent
→ Males and females MUST ALTERNATE.
Pattern: M-F-M-F-M-F... or F-M-F-M-F-M...

Once you know the gender of person at position 1, 
all other genders in all positions are automatically determined!
```

---

### 🚀 TRICK 7 — Two Rows: "Faces" Clue Gives Column

```
"A faces B" in two-row opposite setup:
→ A and B are in the SAME column but different rows.
→ Use this to link Row 1 and Row 2 positions directly.

"A faces B" + "B is 3rd from left in Row 2" → A is at column 3 in Row 1.
```

---

### 🚀 TRICK 8 — South-Facing Left/Right Swap

```
In South-facing arrangements:
"B is to the left of A" from A's perspective (facing South) 
= B is at a HIGHER column number than A in the diagram.

Quick rule: Flip all left/right references for South-facing rows.
Or: Just work from the DIAGRAM'S perspective (physical left/right)
   and convert clue directions accordingly.
```

---

### 🚀 TRICK 9 — Count Available Slots After Placing

```
After each placement, count remaining unplaced persons and remaining empty slots.
They should match! If they don't, you made an error.

Example: 6 persons total. After placing 4, should have 2 unplaced and 2 empty slots.
```

---

### 🚀 TRICK 10 — The "Uniqueness Check"

```
Before finalizing, check that:
1. Each person appears exactly ONCE.
2. Each position is filled by exactly ONE person.
3. ALL clues are satisfied (verify every single clue).

Missing this causes losing all marks even if most clues are right!
```

---

### 🚀 TRICK 11 — Pre-Draw Template

```
For EVERY seating puzzle, the VERY FIRST thing you do:

(a) Count persons (N).
(b) Draw N boxes: [_][_][_]...[_]
(c) Label: position 1, 2, 3... from left.
(d) Write "← LEFT" and "RIGHT →" above.
(e) If two rows: draw both rows with facing directions noted.

This 10-second setup prevents orientation errors throughout.
```

---

### 🚀 TRICK 12 — Negative Clue Technique

```
When stuck (all positive clues exhausted):
For each remaining person, list WHICH positions they CANNOT be in.
The position where ONLY ONE person can go = that person goes there.

Example: 3 remaining persons {D, E, F}, 3 remaining slots {2, 4, 6}.
D cannot be at 4 or 6 → D must be at 2.
Then solve for E and F similarly.
```

---

### 🚀 TRICK 13 — "Immediate Neighbor" Block

```
"A and B sit adjacent to each other" = treat [A][B] or [B][A] as a BLOCK.
Try placing this block at each possible double-position in the row.
Eliminate positions where the block conflicts with other clues.

This "block method" reduces the number of cases to try significantly.
```

---

### 🚀 TRICK 14 — The "Middle" Shortcut

```
For odd N: Middle position = (N+1)/2.
  N=5: middle = 3rd position.
  N=7: middle = 4th position.

For even N: No single middle position (two middle positions: N/2 and N/2+1).

If "A sits at the center" with even N → puzzle may mean "one of the two middle positions."
```

---

### 🚀 TRICK 15 — Verification Always Wins

```
Final step for EVERY puzzle:
Go through each clue one by one and verify your arrangement satisfies it.
Check both the POSITION and the DIRECTION (left/right) carefully.

This catches 90% of errors before submission.
Time spent: 30 seconds. Marks saved: potentially 3–5 questions.
```

## 13. STEP-BY-STEP ATTACK STRATEGY

---

### The Universal 8-Step Method:

```
STEP 1: COUNT total persons. Note if it's 1 row or 2 rows.

STEP 2: DRAW the arrangement framework.
  [_][_][_][_][_][_]
   1  2  3  4  5  6
  ← LEFT           RIGHT →

STEP 3: CATEGORIZE all clues:
  A = Absolute position clues (exact position given)
  R = Relative position clues (position relative to another)
  N = Negative clues (what someone is NOT)
  G = Group/adjacency clues

STEP 4: APPLY clues in order: A → R → G → N

STEP 5: After each placement, CHECK which other clues now become solvable.

STEP 6: If STUCK → try case analysis (try Person X at different positions).

STEP 7: ELIMINATE using negative clues once other placements are done.

STEP 8: VERIFY all clues against final arrangement.
```

---

### Decision Tree When Stuck:

```
STUCK? Try these in order:
│
├─ Is there an "either end" clue? → Try both ends systematically.
│
├─ Is there an "alternating" pattern? (gender/profession) → Apply pattern.
│
├─ Is there a "block" (adjacent pair)? → Try block in each possible location.
│
├─ Are there negative clues unused? → Apply elimination to remaining slots.
│
└─ Are there implications of combinations of clues?
    Example: "A is left of B" + "B is left of C" → A is left of C too.
```

## 14. COMMON MISTAKE ALERTS

---

### ⚠️ MISTAKE 1 — "To the Left" vs "Immediately to the Left"

```
Wrong: "B is to the left of A" → B is immediately next to A on the left.
Right: "B is to the left of A" → B can be ANYWHERE to A's left (not necessarily adjacent).
       "B is IMMEDIATELY to the left of A" → B is exactly 1 position to A's left.

These are very different! Always check for the word "immediate" or "next to."
```

---

### ⚠️ MISTAKE 2 — Between Count Formula Error

```
Wrong: "2 persons between A and B" → |pos_A − pos_B| = 2.
Right: "2 persons between A and B" → |pos_A − pos_B| = 3.

Between = gap − 1. Gap = between + 1. Always add 1!
```

---

### ⚠️ MISTAKE 3 — South-Facing Left/Right Reversal

```
Wrong: Treating all left/right clues as "left = lower position number" regardless of facing.
Right: For SOUTH-facing persons:
       Their LEFT = East = HIGHER position number in diagram.
       Their RIGHT = West = LOWER position number in diagram.

SAFE METHOD: Convert all clues to absolute position numbers before solving.
```

### ⚠️ MISTAKE 4 — "Faces" in Two-Row Questions

```
Wrong: Thinking "A faces B" means A and B are adjacent.
Right: "A faces B" in a two-row opposite setup means A and B are in the SAME COLUMN
       but DIFFERENT rows (directly across from each other).
```

### ⚠️ MISTAKE 5 — Not Checking All Clues at the End

```
Wrong: Declaring the final answer after placing all persons without verifying.
Right: Go through EVERY clue and confirm your arrangement satisfies it.
       Even one unchecked clue can invalidate your entire solution.
```

---

### ⚠️ MISTAKE 6 — "Either End" Meaning

```
Wrong: "P sits at one of the ends" → P is at position 2 or N-1 (near ends).
Right: "P sits at one of the ends" → P is at position 1 (leftmost) OR position N (rightmost).
       "End" = extreme end only.
```

---

### ⚠️ MISTAKE 7 — Confusing "Between" with "Inclusive"

```
Wrong: "D sits between C and G, and is adjacent to C"
       → 3 persons C, D, G counted as adjacent block [C][D][G].
Right: Between C and G means D is in the interior: C...D...G.
       Adjacent to C means D is at C's immediate neighbor position.
       Combined: [C][D][...G...] with D right next to C.
```

---

### ⚠️ MISTAKE 8 — Two-Row "Immediate Left" Direction

```
In two-row setup with Row 1 facing South:
"B is immediately to the left of A in Row 1"
= B is to A's LEFT (from A's view facing South)
= B is one column to the EAST of A
= B is at column (A's column + 1) in the diagram.

This is opposite to what you'd expect from looking at the diagram!
```

---

### ⚠️ MISTAKE 9 — Skipping Negative Clues

```
Wrong: Ignoring "X is not adjacent to Y" until it's needed.
Right: Write down ALL negative clues before starting.
       Apply them AFTER positive clues have narrowed down possibilities.
       Sometimes one negative clue breaks the last ambiguity.
```

---

### ⚠️ MISTAKE 10 — Running Case Analysis Without Exiting Failed Cases

```
Wrong: Starting Case 1 (e.g., A at left), finding contradiction, 
       then continuing to build on the contradicted case.
Right: The moment you find a contradiction in a case, STOP.
       Mark it as ✗ (failed) and immediately move to the next case.
       Don't waste time in a dead-end case.
```

## 🏆 TOP 10 EXAM TIPS

1. **Draw the row BEFORE reading clues** — the moment you see a seating puzzle, draw N boxes and label them 1 to N. This single habit prevents 80% of all errors.

2. **"Immediately" is a magic word** — distinguish between "immediately to the left" (exactly 1 position) and "to the left" (anywhere). Missing this difference costs marks on every puzzle set.

3. **"N persons between" = gap of N+1** — burn this into memory. "2 people between A and B" does NOT mean positions differ by 2; they differ by 3.

4. **Fill exact positions first, relative positions second, elimination last** — this priority order ensures you always have a "home" to work from before applying relative clues.

5. **For "either end" clues, try Left first** — then build the arrangement. If you hit a contradiction, try Right. Rarely will both work for a well-set puzzle.

6. **South-facing = LEFT and RIGHT are SWAPPED** — in a South-facing row, "immediately to your left" = higher position number in the diagram. Always note the facing direction before interpreting left/right clues.

7. **In two-row opposite setups, "faces" = same column** — if A faces B, they share the exact same column number. Use this to instantly link Row 1 and Row 2 positions.

8. **Gender alternating pattern locks in once you know position 1** — if no two same genders can be adjacent and you know position 1's gender, all 8 (or N) genders are immediately determined.

9. **After each placement, scan ALL remaining clues** — whenever you place a person, look through every unused clue to see if any now become directly applicable. Chain placements are faster than starting from scratch.

10. **Always verify every clue at the end** — spend 30 seconds checking each clue against your final arrangement. This catches errors before they cost you marks and can catch an arrangement you built correctly but noted incorrectly.
