## 1. THE BIG PICTURE — WHAT IS CODING & DECODING?

**Coding** = Converting a word/message into a code using a fixed rule.
**Decoding** = Finding the original word/message from the code using the same rule in reverse.

> **Golden Principle:** The SAME rule is always applied consistently. Find the rule → apply it → get the answer.

---

```

## 2. TYPE 1 — LETTER CODING (SHIFT-BASED)

> **Definition:** Each letter of the word is shifted forward (+) or backward (−) by a fixed number of positions in the alphabet.

---

### Sub-Type 1A — Uniform Forward Shift (+n)

Every letter moves forward by the SAME number.

**Example 1:** If CAT = FDW, what is DOG = ?
```
C(3) → F(6)  : +3
A(1) → D(4)  : +3
T(20)→ W(23) : +3
Rule: +3 to every letter
D(4)+3  = G(7)  = G
O(15)+3 = R(18) = R
G(7)+3  = J(10) = J
Answer: GRJ
```

**Example 2:** If BOOK = ERRON, what is PEN = ?
```
B(2) → E(5)  : +3
O(15)→ R(18) : +3
O(15)→ R(18) : +3
K(11)→ N(14) : +3
Rule: +3
P(16)+3 = S(19) = S
E(5)+3  = H(8)  = H
N(14)+3 = Q(17) = Q
Answer: SHQ
```

---

### Sub-Type 1B — Uniform Backward Shift (−n)

**Example 3:** If COME = BNLD, find CODE = ?
```
C(3) → B(2)  : −1
O(15)→ N(14) : −1
M(13)→ L(12) : −1
E(5) → D(4)  : −1
Rule: −1
C(3)−1  = B(2)  = B
O(15)−1 = N(14) = N
D(4)−1  = C(3)  = C
E(5)−1  = D(4)  = D
Answer: BNCD
```

---

### Sub-Type 1C — Different Shift for Each Letter (Increasing/Decreasing)

**Example 4:** If ABCD = BDFH, find EFGH = ?
```
A(1) → B(2)  : +1
B(2) → D(4)  : +2
C(3) → F(6)  : +3
D(4) → H(8)  : +4
Rule: +1, +2, +3, +4 (increasing by 1 each time)
E(5)+1  = F(6)  = F
F(6)+2  = H(8)  = H
G(7)+3  = J(10) = J
H(8)+4  = L(12) = L
Answer: FHJL
```

**Example 5:** If RICE = TKEI, find MILK = ?
```
R(18)→ T(20) : +2
I(9) → K(11) : +2
C(3) → E(5)  : +2
E(5) → I(9)  : +4
Hmm, not uniform. Re-check:
R→T: +2, I→K: +2, C→E: +2, E→I: +4? Let's try odd/even positions:
Position 1(R→T): +2
Position 2(I→K): +2
Position 3(C→E): +2
Position 4(E→I): +4? That's inconsistent.
Actually RICE→TKEI:
R(18)→T(20): +2
I(9)→K(11): +2
C(3)→E(5): +2
E(5)→I(9): +4
Re-examine: Maybe letters are shifted differently.
Try: each letter +2 except last +4? Or:
Check: E→I is actually +4. This might be a +2 uniform shift and the last pair is different. Many exams have such tricks — always verify ALL letters.
Possible rule: +2 for all → MILK: M(13)+2=O, I(9)+2=K, L(12)+2=N, K(11)+2=M → OKNM
```

> **IMPORTANT EXAM TIP:** Always verify the rule with EVERY letter of the given word before applying it.

---

### Sub-Type 1D — Wrap-Around Shift (Circular Alphabet)

When shifting goes past Z, it wraps back to A.

**Example 6:** If Z + 3 = ?
```
Z(26) + 3 = 29
29 − 26 = 3 = C
Z + 3 = C (wraps around)
```

**Example 7:** If XYZ is coded as ABB... (wait):
```
X(24)+3 = 27 = A? No: 27−26=1=A, but 27th position = A after wrap. Hmm:
Correct wrap formula: If result > 26, subtract 26.
X(24)+3 = 27 → 27−26 = 1 = A
Y(25)+3 = 28 → 28−26 = 2 = B
Z(26)+3 = 29 → 29−26 = 3 = C
XYZ coded as ABC (with +3 and wrap)
```

---

### Sub-Type 1E — Alternate Shift (Different rules for odd/even positions)

**Example 8:** If TEAM = VGCO, find WORK = ?
```
T(20)→V(22): +2 (position 1, odd)
E(5)→G(7):   +2 (position 2, even)
A(1)→C(3):   +2 (position 3, odd)
M(13)→O(15): +2 (position 4, even)
All +2. Uniform.
WORK: W(23)+2=Y, O(15)+2=Q, R(18)+2=T, K(11)+2=M → YQTM
```

**Example 9:** If FLOWER = GKPYFX, find CLOCK = ?
```
F(6) → G(7)  : +1
L(12)→ K(11) : −1
O(15)→ P(16) : +1
W(23)→ Y(25) : +2? No, let's recount:
F→G: +1, L→K: −1, O→P: +1, W→Y: +2? Hmm.
Wait: FLOWER has 6 letters, GKPYFX has 6 letters.
F(6)→G(7): +1
L(12)→K(11): −1
O(15)→P(16): +1
W(23)→Y(25): +2? Or check again: W→Y skip one... 
Actually: odd positions +1, even positions −1:
F(pos1,odd)→G: +1 ✓
L(pos2,even)→K: −1 ✓
O(pos3,odd)→P: +1 ✓
W(pos4,even)→... Y? W=23, 23−1=22=V, not Y. 
Re-examine: F→G(+1), L→K(−1), O→P(+1), W→X(−1)? W(23)−1=22=V, not X.
Hmm, GKPYFX: G,K,P,Y,F,X
W→Y: 23→25: +2. E→F: 5→6: +1. R→X: 18→24: +6? That's not consistent.
Likely exam error OR rule is: +1,−1,+1,+2,+1,+6 — which means it's NOT a simple shift.
KEY LESSON: If pattern breaks, the question might involve a different type (like reverse+shift).
```

> **LESSON:** When a pattern seems inconsistent, try: reverse the word first, THEN shift. Or shift, THEN reverse.

## 3. TYPE 2 — NUMBER CODING

> **Definition:** Letters are replaced by numbers following a specific rule — could be their position, reverse position, or a custom mapping.

---

### Sub-Type 2A — Position-Based (A=1, B=2... Z=26)

**Example 10:** If CAT = 3120, find DOG = ?
```
C=3, A=1, T=20 → 3,1,20 → written as "3120"
D=4, O=15, G=7 → "4157"
Answer: 4157
```

**Example 11:** If BALL = 2+1+12+12 = 27, find CAT = ?
```
Sum of positions: C(3)+A(1)+T(20) = 24
Answer: 24
```

---

### Sub-Type 2B — Reverse Position (Z=1, Y=2... A=26)

**Example 12:** If CAT = 24, 26, 7 — find DOG = ?
```
Reverse position:
C → 27−3 = 24 ✓
A → 27−1 = 26 ✓
T → 27−20 = 7 ✓
Rule: Reverse position = 27 − forward position
D → 27−4 = 23
O → 27−15 = 12
G → 27−7 = 20
Answer: 23, 12, 20
```

---

### Sub-Type 2C — Custom Number Mapping

**Example 13:** If A=2, B=3, C=4... (each letter = position+1), find CAT = ?
```
C = 3+1 = 4
A = 1+1 = 2
T = 20+1 = 21
Answer: 4, 2, 21
```

**Example 14:** If A=1, E=2, I=3, O=4, U=5 (vowels only coded), find the code for AUDIT:
```
A=1, U=5, D=not coded, I=3, T=not coded
Vowels in AUDIT: A(1), U(5), I(3) → Code = 153 (for vowels only)
```

---

### Sub-Type 2D — Numerical Operations on Position

**Example 15:** If COLD = 3, 15, 12, 4 → sum = 34. What word has code 52?
```
Find a word where sum of letter positions = 52.
This type requires trial: e.g., LOVE = 12+15+22+5 = 54 (no), 
LONG = 12+15+14+7 = 48 (no)... keep trying or work backwards.
Exam shortcut: usually the options are given — just calculate each.
```

**Example 16:** If A=1, B=2...Z=26, and each letter code = position × 2:
```
A=2, B=4, C=6, D=8...
CAT = 6, 2, 40
```

---

### Sub-Type 2E — Number represents RANK from end

**Example 17:** Find: KING → code using reverse rank (A=26, B=25, Z=1):
```
K = 27−11 = 16
I = 27−9 = 18
N = 27−14 = 13
G = 27−7 = 20
Code = 16, 18, 13, 20
```

## 4. TYPE 3 — SUBSTITUTION CODING

> **Definition:** A word (or object/place/person) is called by a different name. You must identify what the new name refers to.

---

### How It Works:

Given a set of substitution rules, answer questions about what something is "called."

**Example 18:**
```
"Sky" is called "Sea"
"Sea" is called "River"
"River" is called "Mountain"
"Mountain" is called "Road"
"Road" is called "Forest"

Question: Where do fish live?
Step 1: Fish actually live in "Sea."
Step 2: "Sea" is called "River."
Answer: River
```

**Example 19:**
```
"White" is called "Blue"
"Blue" is called "Red"
"Red" is called "Black"
"Black" is called "Violet"
"Violet" is called "Orange"

Question: What is the color of blood?
Step 1: Blood is actually "Red."
Step 2: "Red" is called "Black."
Answer: Black
```

**Example 20:**
```
"Book" is called "Pen"
"Pen" is called "Eraser"
"Eraser" is called "Sharpener"
"Sharpener" is called "Scale"

Question: With what do you write?
Step 1: You write with "Pen."
Step 2: "Pen" is called "Eraser."
Answer: Eraser
```

---

### ⚡ TRICK for Substitution:
```
Step 1: Identify the REAL answer to the question.
Step 2: Find that real answer in the LEFT side of substitution rules.
Step 3: The RIGHT side of that rule = your final answer.
```

## 5. TYPE 4 — REVERSE CODING

> **Definition:** Letters of a word are reversed (or coded using their mirror/opposite in the alphabet).

---

### Sub-Type 4A — Word Reversed

**Example 21:** If CAT = TAC, find DOG = ?
```
Rule: Reverse the letters of the word.
DOG reversed = GOD
Answer: GOD
```

**Example 22:** If STRONG = GNORTS, find MARKET = ?
```
Rule: Reverse the word.
MARKET reversed = TEKRAM
Answer: TEKRAM
```

---

### Sub-Type 4B — Each Letter Replaced by Opposite (Mirror) Letter

**Example 23:** If A=Z, B=Y, C=X... (opposites), find CAT = ?
```
C(3) → opposite = 27−3 = 24 = X
A(1) → opposite = 27−1 = 26 = Z
T(20)→ opposite = 27−20 = 7 = G
Answer: XZG
```

**Example 24:** If COME = XLNV, find BACK = ?
```
C(3)→X(24): 3+24=27 ✓ (opposite)
O(15)→L(12): 15+12=27 ✓
M(13)→N(14): 13+14=27 ✓
E(5)→V(22): 5+22=27 ✓
Rule: Replace each letter with its alphabet opposite
B(2)→Y(25): 2+25=27 ✓
A(1)→Z(26)
C(3)→X(24)
K(11)→P(16): 11+16=27 ✓
Answer: YZXP
```

---

### Sub-Type 4C — Reverse + Shift Combined

**Example 25:** If WATER = HVGVI, find FIRE = ?
```
Reversed WATER = RETAW
R(18)→H(8)? 18→8: −10? Not clean.
Try opposite: W→H: W(23)+H(8)=31 ≠ 27. Hmm.
Try: Reverse word first → RETAW
R→H: opposite? R(18)→I(9): 18+9=27 ✓ YES!
E(5)→V(22): 5+22=27 ✓
T(20)→G(7): 20+7=27 ✓
A(1)→Z(26): 1+26=27 ✓
W(23)→D(4): 23+4=27 ✓
Rule: Reverse word, then replace each with opposite letter.
FIRE reversed = ERIF
E(5)→V(22)
R(18)→I(9)
I(9)→R(18)
F(6)→U(21)
Answer: VIRU
```

> **Key Insight:** Many hard questions combine TWO operations. Common combinations:
> - Reverse word + shift each letter
> - Shift each letter + reverse word
> - Replace with opposite + shift

## 6. TYPE 5 — MIXED LETTER & NUMBER CODING

> **Definition:** The code contains both letters and numbers, with each letter replaced by a number AND/OR each number replaced by a letter.

---

**Example 26:** If PAINT = P2N20, find BRUSH = ?
```
P → P (unchanged)
A(1) → 2? Hmm, that's A+1=2.
I → N? That doesn't make sense numerically.
Wait: P-A-I-N-T:
P stays P
A(1) → 2 (position+1)
I → N: I is position 9, N is position 14: +5? Or I is replaced by next vowel?
N → 2? N is position 14, 2 ≠ 14.
Re-read: PAINT = P2N20
P=P, A=2, I=I→ wait, maybe vowels become numbers:
A(1st vowel)=1? But it says 2. 
Or: Vowels replaced by their position number, consonants kept:
A(position 1 in alphabet)=1? But shows 2.
Hmm. Maybe: VOWELS replaced by (position in alphabet)÷something?
Let me try: Vowels get their serial number among vowels: A=1st vowel→1, E=2, I=3, O=4, U=5.
PAINT: P(cons), A(1st vowel→1), I(3rd vowel→3), N(cons), T(cons) → P1N3T? Not matching P2N20.
Different approach: consonants kept, vowels get position value:
A=1, I=9, but shows 2 and nothing for I... 
Most likely the question code as given may use: consonants stay, vowels → position: A=1→shown as 2 might mean +1 applied. I=9, so shows as... wait it shows N not 9.
THIS ILLUSTRATES: always be given a CLEAR example in real exams. Let me provide a clean example:
```

**Example 26 (Clean):** If CRANE = C18135, find TRAIN = ?
```
C → C (consonant, kept as letter)
R → 18 (R's position: 18, replaced by number)
A → 1 (A's position: 1, replaced by number)
N → N (consonant, kept)
E → 5 (E's position: 5, replaced by number)
Rule: Vowels → their alphabet position number; Consonants → kept as letter
TRAIN:
T → T
R → 18
A → 1
I → 9
N → N
Answer: T18A9N → Wait, A is a vowel: T(cons)→T, R(cons)→R, A(vowel)→1, I(vowel)→9, N(cons)→N
Answer: TR19N → Actually: T, R, 1, 9, N → TR19N
```

**Example 27:** If SKY = 19K25, find SUN = ?
```
S(19) → 19 (replaced by position number)
K → K (kept as letter)
Y(25) → 25 (replaced by position number)
Rule: Consonants → their position number, vowels → kept as letter
SUN:
S(19) → 19
U → U (vowel, kept)
N(14) → 14
Answer: 19U14
```

## 7. TYPE 6 — SYMBOL / FICTITIOUS LANGUAGE CODING

> **Definition:** Letters or words are replaced by symbols (@, #, *, !, etc.) or by letters from a fictitious/made-up language. Given a translation, decode the unknown.

---

### Sub-Type 6A — Symbol Substitution

**Example 28:**
```
If A=@, B=#, C=*, D=&, E=$
Find the code for BEAD:
B=# E=$ A=@ D=&
Answer: #$@&
```

**Example 29:**
```
If @=#, $=*, &=!, and #=@:
Note: This is a circular substitution. Track carefully.
@ → # → @ (circular pair)
$ → * (not circular here)
& → !
```

---

### Sub-Type 6B — Fictitious Language (Word-to-Word mapping)

**Example 30:**
```
In a certain code language:
"sit kar lo" means "please sit down"
"lo bat kar" means "please talk now"
"bat hum sit" means "we talk sitting"

Find the code for "now":
"sit kar lo" → please, sit, down
"lo bat kar" → please, talk, now
"bat hum sit" → we, talk, sitting

"lo" appears in sentences 1 & 2 → maps to "please"
"kar" appears in sentences 1 & 2 → maps to... sit/down (sentence1) and talk/now (sentence2) → "kar"≠now
"bat" appears in sentences 2 & 3 → maps to "talk"
"sit" appears in sentences 1 & 3 → maps to "sit" or "sitting"
"lo" = "please" (confirmed: appears where "please" appears)
"kar" in sentence1: remaining = sit/down; "kar" in sentence2: remaining after "bat"=talk → "lo"=please, "bat"=talk, so "kar" maps to... 
Sentence 2: "lo bat kar" = "please talk now" → lo=please, bat=talk → kar=now
Answer: kar
```

---

### Sub-Type 6C — Three-Statement Word Decoding Method

**THE MOST COMMON EXAM TYPE. Master this method:**

```
STEP-BY-STEP METHOD:
1. Take two sentences that share EXACTLY ONE common code word.
2. The shared code word = shared English word between those two sentences.
3. Eliminate and repeat for all words.

Example:
"sa re ga" = "she is good"
"sa ma pa" = "she goes home"
"re pa la" = "is she going"

From sentence 1 & 2: "sa" is common → "she" is common → sa = she
From sentence 1 & 3: "re" is common → "is" is common → re = is
From sentence 2 & 3: "pa" is common → "goes/going" is common → (goes or going)
```

## 8. TYPE 7 — SENTENCE / WORD CODING

> **Definition:** Entire words or sentences are coded by rearranging letters, changing positions, or applying word-level operations.

---

### Sub-Type 7A — Letters Within Word Rearranged

**Example 31:** If ARMY is coded as AMRY, find NAVY = ?
```
A R M Y → A M R Y
Compare: A stays, R&M swapped, Y stays.
Rule: Middle two letters are swapped.
N A V Y → N V A Y
Answer: NVAY
```

**Example 32:** If HOTEL is coded as LEOTH, find POWER = ?
```
H O T E L → L E O T H
Original: H(1) O(2) T(3) E(4) L(5)
Coded:    L(5) E(4) O(3... wait): L E O T H
Positions in code: L=5th, E=4th, O=2nd, T=3rd, H=1st
Rule: Reversed! HOTEL reversed = LETOH. But code is LEOTH.
Hmm: LETOH ≠ LEOTH. Close but not exact.
Try: first letter goes to end, rest shift left:
H O T E L → O T E L H? No.
Try: Last 2 letters moved to front:
EL + HOT = ELHOT? No.
Try: word split in half and swapped:
HOT + EL = EL + HOT = ELHOT? No.
Try: reverse pairs: (HO)(TE)(L) → (OH)(ET)(L) = OHETL? No.
LEOTH: L=last, E=4th, O=2nd, T=3rd, H=1st → positions 5,4,2,3,1.
Pattern: 5,4,2,3,1 applied to POWER(P,O,W,E,R):
P=1, O=2, W=3, E=4, R=5
Code positions 5,4,2,3,1 → R, E, O, W, P → REOWP
Answer: REOWP
```

---

### Sub-Type 7B — Letters Shifted Within Word Based on Position

**Example 33:** If 'MIND' is coded as 'KGLB', find 'BODY' = ?
```
M(13)→K(11): −2
I(9)→G(7):   −2
N(14)→L(12): −2
D(4)→B(2):   −2
Rule: each letter −2
B(2)−2=26=Z (wrap: 2−2=0→0+26=26=Z)
O(15)−2=M(13)
D(4)−2=B(2)
Y(25)−2=W(23)
Answer: ZMBW
```

---

### Sub-Type 7C — First and Last Letters Swapped / Specific Position Operations

**Example 34:** If CLOUD is coded as DLUOC, find STORM = ?
```
C L O U D → D L U O C
C↔D? No, C(1st)→D(last shifted?):
Original: C(1) L(2) O(3) U(4) D(5)
Coded:    D(1) L(2) U(3) O(4) C(5)
Compare: 1st letter D = original 5th letter ✓
         2nd letter L = original 2nd letter ✓
         3rd letter U = original 4th letter ✓
         4th letter O = original 3rd letter ✓
         5th letter C = original 1st letter ✓
Rule: First and last letters swap, middle letters reverse (3rd&4th swap, 2nd stays):
Actually: positions 1,2,3,4,5 → become positions 5,2,4,3,1
STORM: S(1) T(2) O(3) R(4) M(5)
Apply rule (5,2,4,3,1): M, T, R, O, S → MTROS
Answer: MTROS
```

## 9. TYPE 8 — CONDITIONAL / RULE-BASED CODING

> **Definition:** Specific conditions modify how letters are coded. Common in Bank exams (SBI PO, IBPS PO).

---

### The Classic Format:

Letters have digit codes. Special conditions apply based on:
- Whether the first/last letter is a vowel or consonant
- Whether letters satisfy some property

**Example 35:**
```
Letter Code Table:

## 12. MASTER TRICKS & SHORTCUTS

---

### 🚀 TRICK 1 — EJOTY for Instant Position Finding
```
Forward:  E=5, J=10, O=15, T=20, Y=25
Reverse:  V=5, Q=10, L=15, G=20, B=25
Any letter's position found in under 2 seconds!
Example: What position is R? 
R is 3 after O(15) → 15+3=18. R=18.
```

### 🚀 TRICK 2 — Opposite Letter = 27 − Position
```
Opposite of any letter X = the letter at position (27 − pos(X))
A(1)↔Z(26), B(2)↔Y(25), C(3)↔X(24)... M(13)↔N(14)
Quick: Opposite of T(20) = 27−20 = 7 = G ✓
```

### 🚀 TRICK 3 — Shift Forward vs Backward Quick Conversion
```
+n shift forward = −(26−n) shift backward
Example: +3 forward = −23 backward (both give same result circularly)
Easier: just add n and if >26, subtract 26.
```

### 🚀 TRICK 4 — Always Identify Rule from GIVEN Example First
```
Never guess. Always:
1. Take the GIVEN coded word.
2. Write original letters and their codes side by side.
3. Find the mathematical relationship.
4. VERIFY with every letter.
5. THEN apply to the asked word.
```

### 🚀 TRICK 5 — Substitution: Answer the Question First, Then Substitute
```
"What do fish live in?" → answer is "Sea" (real world)
"Sea" is called "River" → answer = River
Don't get confused: find REAL answer FIRST, then look it up in the substitution table.
```

### 🚀 TRICK 6 — Fictitious Language: Common Word = Common Code
```
If "ta ma" = "good morning" and "ta pa" = "good evening":
"ta" is common in both sentences AND "good" is common in both meanings.
Therefore ta = good.
This works when EXACTLY ONE word is common.
```

### 🚀 TRICK 7 — Wrong Coding Detection
```
If asked which word is wrongly coded:
1. Identify the coding rule from the correctly coded words.
2. Apply rule to each option.
3. The one that doesn't match = wrongly coded.
```

### 🚀 TRICK 8 — Conditional Coding: Check Ends First
```
In rule-based coding:
ALWAYS identify first letter (vowel/consonant?) and last letter (vowel/consonant?) BEFORE starting.
This determines which condition applies.
Middle letters ALWAYS follow the normal code table.
```

### 🚀 TRICK 9 — Matrix Coding: Locate then Record
```
Don't try to remember where letters are.
Systematically scan row 0, then row 1, etc.
Write (row, col) immediately when found.
For 5×5 matrix: 25 letters fit perfectly (A–Y or custom set).
```

### 🚀 TRICK 10 — Wrap-Around Formula
```
If shift result > 26: subtract 26
If shift result < 1: add 26
Example: Z(26)+3 = 29 → 29−26 = 3 = C
Example: A(1)−3 = −2 → −2+26 = 24 = X
```

### 🚀 TRICK 11 — Reverse + Shift: Two-Operation Combo
```
When simple shift doesn't work, try:
Option A: Reverse the word → then shift each letter
Option B: Shift each letter → then reverse the word
Both give different results. Test both when stuck.
```

### 🚀 TRICK 12 — Number Coding Verification
```
When letters coded as numbers:
Sum of codes / product / specific pattern?
Always check: is it position, position+1, position−1, position×2, etc.?
Check 3 letters before concluding the rule.
```

### 🚀 TRICK 13 — Consonant/Vowel Quick Test
```
Vowels: A, E, I, O, U (only 5!)
Everything else = consonant (21 consonants)
Quick check: Is the letter A/E/I/O/U? No → it's a consonant.
```

### 🚀 TRICK 14 — Letter Position from End
```
Position from end = 27 − position from beginning
OR: count backwards: Z=1, Y=2, X=3...
Useful when code uses position from end.
```

### 🚀 TRICK 15 — Odd/Even Position Trick
```
Some codes apply different rules for odd/even positions.
Number odd positions: 1st, 3rd, 5th letters of word
Number even positions: 2nd, 4th, 6th letters of word
If pattern seems inconsistent → CHECK if it's odd/even split!
```

## 13. STEP-BY-STEP ATTACK STRATEGY

---

### When you see a Coding & Decoding question:

```
STEP 1: READ the given example carefully.
        (e.g., "If CAT = FDW, find DOG = ?")

STEP 2: IDENTIFY the type:
        → Letters? → Letter coding
        → Numbers only? → Number coding  
        → Mixed? → Mixed type
        → Symbols given? → Symbol/Fictitious
        → Conditions given? → Conditional coding

STEP 3: DECODE the rule:
        → Write each letter of original word
        → Write corresponding code below it
        → Find the mathematical/logical difference
        → Verify with ALL letters, not just first one

STEP 4: APPLY the rule to the asked word:
        → Write each letter of the asked word
        → Apply the identified rule to each letter
        → Write down the result

STEP 5: VERIFY:
        → Does your answer look consistent with the format of the given code?
        → Same number of characters? Same type (letters/numbers)?
        → Cross-check 1-2 letters manually.
```

---

### Troubleshooting When Pattern Seems Wrong:

```
Pattern breaks? Try these in order:
□ Is it +n or −n shift? (check all letters)
□ Is it opposite letter? (sum = 27?)
□ Is it reverse of word? (check letter positions)
□ Is it reverse word + shift?
□ Is it shift + reverse?
□ Is it odd/even position split?
□ Is it two separate codes interleaved?
□ Is it position-based number? (A=1...Z=26)
□ Is it reverse position? (A=26...Z=1)
```

## 14. COMMON MISTAKE ALERTS

---

### ⚠️ MISTAKE 1 — Not Verifying Rule With All Letters
```
Wrong: Check only 1-2 letters then assume rule.
Right: ALWAYS verify with EVERY letter of the example word.
Many "tricks" are hidden in the last letter!
```

### ⚠️ MISTAKE 2 — Confusing +shift and −shift
```
If original → code looks larger → it's a + shift (forward)
If original → code looks smaller → it's a − shift (backward)
Example: C→F is +3 (C=3, F=6, 6−3=+3)
Example: F→C is −3 (F=6, C=3, 3−6=−3)
```

### ⚠️ MISTAKE 3 — Forgetting Wrap-Around
```
Wrong: Y+3 = AB (not a valid letter!)
Right: Y(25)+3 = 28 → 28−26 = 2 = B
Always: if position > 26, subtract 26.
```

### ⚠️ MISTAKE 4 — Substitution Type: Answering Before Substituting
```
Wrong: "Fish live in sea" → answer = sea
Right: "Fish live in sea" → "sea" is called "river" → answer = river
Always substitute the real answer using the given rules!
```

### ⚠️ MISTAKE 5 — Opposite Letter Mistake
```
Opposite of A is Z (not Y, not B)
Formula: Opposite of letter at position n = letter at position (27−n)
A(1) → Z(26): 1+26=27 ✓
B(2) → Y(25): 2+25=27 ✓
M(13)→ N(14): 13+14=27 ✓
```

### ⚠️ MISTAKE 6 — Conditional Coding: Applying Condition to Middle Letters
```
Wrong: Applying the special code (* or # or $ or @) to ALL letters.
Right: Special codes apply ONLY to the first and last letters.
Middle letters ALWAYS get their normal digit code.
```

### ⚠️ MISTAKE 7 — Fictitious Language: Using a Word That Appears in More Than One Sentence Pair
```
If a code word appears in ALL three sentences → it maps to the common English word.
If you use a word appearing in only one sentence → you can't isolate its meaning.
Always find words that appear in EXACTLY 2 sentences to isolate meaning.
```

### ⚠️ MISTAKE 8 — Matrix Coding: Row, Column Order
```
Wrong: Writing (column, row)
Right: ALWAYS write (row, column) — row first, then column.
Row = horizontal (↕), Column = vertical (↔)
```

### ⚠️ MISTAKE 9 — Assuming All Letter Codes Are Same-Direction Shifts
```
Some questions use +n for consonants and −n for vowels (or vice versa).
Some use +n for odd positions and +m for even positions.
Don't assume uniform shift until verified!
```

### ⚠️ MISTAKE 10 — Number Coding: Not Distinguishing Between Position and Reverse Position
```
A=1 (position) vs A=26 (reverse position) — HUGE difference!
Test: Check if the number for A is 1 (position) or 26 (reverse position).
This single check tells you which system is used.
```

## 🏆 TOP 10 EXAM TIPS

1. **EJOTY is your best friend** — memorize E=5, J=10, O=15, T=20, Y=25 and find any letter's position instantly.

2. **Always verify the rule with EVERY letter** — never assume a rule from just one or two letters.

3. **Opposite = 27 minus position** — A↔Z, B↔Y, M↔N. This is the most common hidden rule.

4. **For substitution: answer the real question first, THEN substitute** — don't substitute, then answer.

5. **In fictitious language: find sentences with exactly 1 common word to isolate meanings** — start with the pair sharing the most words.

6. **Conditional coding: check the first AND last letter before touching any other letter** — middle letters are always coded normally.

7. **Matrix coding: always write (row, column) — row comes FIRST** — most mistakes happen here.

8. **When shift doesn't work, try: reverse word + shift, OR shift + reverse word** — many hard questions combine two steps.

9. **Wrap around carefully** — Z+1=A, A−1=Z. Use: if result>26, subtract 26; if result<1, add 26.

10. **In number coding: always check if it's forward position (A=1) or reverse position (A=26)** — checking just the code for A or Z immediately tells you which system it is.
