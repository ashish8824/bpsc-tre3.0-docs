### 4.1 UNIT 1: OOP & C++ PROGRAMMING
**Weight: ~20% of CS paper = ~16 questions**

#### A. Object-Oriented Programming Concepts
- Four pillars: **Encapsulation, Inheritance, Polymorphism, Abstraction**
- Class vs Object — definition and difference
- Instance vs Class variables
- Constructor types: Default, Parameterized, Copy
- Destructor — when called, purpose
- **Abstract class** — must have at least one pure virtual function
- **Virtual function** vs **Pure virtual function**
- **Virtual inheritance** — avoids diamond problem
- Access specifiers: public, private, protected
- Difference between **struct** and **class** in C++ (default access: public vs private)
- Friend functions and friend classes
- Operator overloading
- Runtime vs Compile-time polymorphism
- Method overloading vs Method overriding
- Inheritance types: Single, Multiple, Multilevel, Hierarchical, Hybrid
- **Correct syntax of inheritance**: `class derived : access base {}`
- `extends` keyword in Java (OOP concept)

#### B. C++ Language Specifics
- **Variable naming rules** — cannot start with digit (e.g., 123Variable is invalid)
- Header file extensions: `.h` (user-defined), `.cpp` (source)
- `NULL` pointer deletion — `delete ptr` where ptr=NULL is valid (no crash)
- `delete` vs `delete[]` — single object vs array
- `new` vs `malloc` — operator vs function; constructor invocation
- `ios::trunc` mode — truncate file to zero
- `ios::in`, `ios::out` — input/output file modes; can be combined
- File opening modes combination in `open()`
- Shortcut keys: Ctrl+F9 = compile and run in Turbo C++
- Semicolon missing error message
- String concatenation with char arrays — compile error (cannot use + directly)
- User-defined header files use `.h` extension
- `is_array()` function — checks if variable is an array type
- `rank()` — returns dimension of array
- Array declaration: `int array[10];` is correct
- Scope of variable inside user-defined function
- **Output prediction questions** — recursive functions, pointer arithmetic
- Exception handling: `try`, `catch`, `throw` blocks
- Inline functions — inserted at call point, NOT large/complex

#### C. Code Output Questions (Practice these!)
- Recursive factorial with wrong condition (a+1 instead of a-1) → segmentation fault
- NULL pointer delete → compiles and runs successfully
- Char array string concatenation → compile error
- Bitwise OR + bool arithmetic
- `sizeof()` — does NOT evaluate expression

### 4.2 UNIT 2: DATA STRUCTURES & ALGORITHMS
**Weight: ~18% of CS paper = ~14 questions**

#### A. Arrays
- Definition — container of similar type elements
- Random access using index — main advantage
- Index starts from 0; nth element = array[n-1]
- Types in C++: Single-dimension, Multi-dimension
- Sparse Matrix — zero elements > (m*n)/2
- Array highly used in: Spatial locality

#### B. Stacks
- LIFO principle
- Applications: **String reversal, Backtracking, Expression evaluation, Recursion**
- NOT an application: Asynchronous data transfer (that's Queue)
- Infix to Postfix conversion uses Stack — **Time complexity O(N)**
- Infix to Prefix also uses Stack

#### C. Queues
- FIFO principle
- Check **underflow** before deletion from queue
- Check **overflow** before insertion
- Circular Queue — also called **Ring buffer**
- Reason for circular queue — **avoid wastage of memory**
- Circular queue empty condition: `front = rear = -1`
- Overflow in linear queue: `rear = MAX_SIZE - 1`
- Deque (Double-ended queue) — insert/delete from both ends
- Priority Queue — best implemented with **Binary Heap**
- NOT a Queue application: Balancing of symbols (that's Stack)
- Deque with singly linked list — delete from rear is **O(N)**

#### D. Linked Lists
- Singly vs Doubly linked list
- Doubly LL: bidirectional traversal, extra space, **HARDER** to implement than singly
- Circular linked list

#### E. Trees
- Binary Tree traversals: **Preorder, Inorder, Postorder** (NOT Randomized traversal)
- Binary Search Tree (BST):
  - Left child < root < right child
  - Left and right subtrees are also BSTs
- BST insertion and deletion
- Given preorder traversal → derive postorder (practice this!)
- Number of distinct BSTs from n keys = **Catalan number Cₙ**
  - C₄ = 14
- Red-Black Tree: new root node = Black; new non-root = Red
- Balanced tree: height difference between subtrees ≤ 1
- Spanning tree with N vertices has **N-1 edges**

#### F. Algorithms & Complexity
- Big-O notation = **Upper bound** of algorithm's runtime
- Time complexities:
  - Binary search: O(log N)
  - Infix to postfix: O(N)
  - Merge sort: O(N log N)
  - Bubble sort: O(N²)
- Sorting paradigms:
  - **Divide and Conquer**: Merge Sort, Quick Sort
  - Heap Sort: uses heap data structure
  - Bubble Sort: comparison-based
- Divide and Conquer steps: Divide, Conquer, Combine
- Hash Table — best for efficient search, insert, delete
- NP-Complete vs Polynomial:
  - Shortest Path = **polynomial** (NOT NP-Complete)
  - Travelling Salesman = NP-Complete
- Graph traversal: BFS, DFS (Greedy/DP/D&C are algorithm paradigms, not traversals)

### 4.3 UNIT 3: DBMS & SQL
**Weight: ~12% of CS paper = ~10 questions**

#### A. Database Concepts
- Database = organized collection of data that can be accessed, updated, managed
- Types: Relational, Network, Hierarchical, Distributed (NOT "Decentralized" as a type)
- NOT a DBMS: Google (it's a search engine)
- DBMS features: minimum redundancy, high security, multi-user access (NOT single-user only)
- DBMS NOT a feature: single-user access only
- Metadata = information about data
- DBMS acts as interface between: **database application and the database**
- **Superkey** = set of attributes that uniquely identify a record
- Primary key = uniquely identifies each record; only **1** per table
- Foreign key = references primary key of another table
- Referential integrity constraint

#### B. SQL Commands
- SQL Views = **Virtual tables**
- `ORDER BY` default = **ASC**
- `HAVING` clause = like WHERE but for **groups** (not rows)
- `WITH` clause = creates **temporary relation**
- `COMMIT` = makes updates permanent
- `ROLLBACK` = undoes transaction
- `TRUNCATE` vs `DELETE` difference
- `ROUND(65.726, -1)` = **70** (rounds to nearest 10)
- Not a constraint in SQL: **UNION** (PRIMARY KEY, NOT NULL are constraints)
- Not a valid aggregate function: **COMPUTE** (COUNT, SUM, AVG, MAX, MIN are valid)
- DELETE from relation: `DELETE FROM teaches;`
- SQL hierarchy: environments contain **catalogs**, catalogs contain **schemas**
- After GROUP BY, use **HAVING** for predicates with aggregate functions
- `db_accessadmin` role — for adding/removing user IDs

#### C. Normalization
- 2NF = eliminates partial dependencies (full functional dependency on PK)
- 3NF = adequate for normal RDBMS design
- Data in two places → **wasted space AND data inconsistency** (both)
- RAW data type — stores unstructured data

### 4.4 UNIT 4: COMPUTER NETWORKS
**Weight: ~10% of CS paper = ~8 questions**

#### A. OSI & TCP/IP Models
- OSI has 7 layers; TCP/IP has 4 layers
- TCP/IP "Host-to-Host" layer corresponds to OSI **Transport layer**
- Data Link layer sublayers:
  - **LLC (Logical Link Control)** — upper sublayer
  - **MAC (Media Access Control)** — performs functions depending on medium type
- Network layer — routing and forwarding

#### B. IP Addressing
- IPv4 = 32 bits
- IPv6 = **128 bits** (NOT 32, 64, or 256)
- IPv6 header translation: flow label is **considered** (NOT discarded)
- Memory-mapped I/O — I/O and memory **share same address space**

#### C. Protocols
- SMTP — sending email; when mail server sends to another server → **SMTP client**
- TCP — connection-oriented, three-way handshake, single stream
- Ping — summarizes packet loss and round-trip delay (NOT "Packet Internet Generator" — common wrong option)
- MQTT — IoT device communication protocol
- TDM — slots divided into **frames**
- Socket = endpoint of inter-process communication flow

#### D. Network Topologies & Devices
- Star topology requires central controller/hub
- NOT a topology: **Peer-to-peer** (it's a network type, not topology)
- WWW = collection of hyperlinked documents on Internet
- Switch — connects devices in LAN
- Router — connects multiple networks together
- Subnet mask — identifies **network portion** of IP address

#### E. Transmission Media
- Optical fiber — highest transmission speed; best for electrical interference
- UTP (Unshielded Twisted Pair) — vulnerable to electrical interference
- Coaxial cable — moderate
- Wireless: radio waves, microwaves, infrared
- Sniffers prevented by: **switched network**

#### F. Security
- Asymmetric key cryptography — private key kept by **receiver**
- System integrity breach: **Full access rights for all users**

### 4.5 UNIT 5: DIGITAL ELECTRONICS & LOGIC GATES
**Weight: ~15% of CS paper = ~12 questions (growing topic)**

#### A. Logic Gates
- **Universal gates: NAND and NOR** (both can implement any Boolean function)
- Basic gates: AND, OR, NOT, XOR, XNOR
- XNOR — provides HIGH output for **same inputs**
- XOR — provides HIGH output for **different inputs**
- Half adder outputs: **Sum and Carry**
- Seven-segment decoder — used to display time at railway stations

#### B. Boolean Algebra
- Boolean algebra applications: designing digital computers, building logic symbols, circuit theory (**all three** — answer is D)
- De Morgan's theorem — used for simplifying Boolean expressions
- SOP (Sum of Products) form: Y = AB + BC + AC
- POS (Product of Sums)
- Canonical forms: **SOM (Sum of Minterms) and POM (Product of Maxterms)**
- Dual of Boolean expression — interchange 0s and 1s, AND and OR operators
- Boolean expression simplification — practice K-Map

#### C. Number Systems
- Decimal 1234:
  - Binary = 10011010010
  - Octal = 2322
  - Hex = 4D2
- Binary 1011.1101 = 11.8125 decimal
- Max BCD digit value = **9** (NOT 15; BCD only uses 0–9)
- Booth's algorithm — used for **binary multiplication**

#### D. Memory & Sequential Circuits
- Types of ROM: **MROM, PROM, EPROM** (answer = More than one)
- PLA = **Programmable Logic Array**
- Flip-flop: negative level triggered → changes state when clock is **negative**
- Error detection codes: Hamming, CRC, Checksum (all three)
- Select lines for 8:1 MUX = **3** (2³ = 8)
- Truth table rows for 4-input OR gate = **16** (2⁴)

### 4.6 UNIT 6: OPERATING SYSTEMS
**Weight: ~12% of CS paper = ~10 questions (fast-growing topic)**

#### A. Memory Management
- Types of memory management: Paging, Segmentation, Virtual Memory
- **Compaction** — shifts processes to make them contiguous; all free memory in one block
- **Fragmentation** — the PROBLEM (internal and external)
- External fragmentation — free memory exists but is non-contiguous
- Paging:
  - Page = fixed-size block of virtual memory
  - Page offset bits for 4KB page = **12 bits** (2¹² = 4096 = 4KB)
- **TLB (Translation Lookaside Buffer)** — organized as associative cache
- Segmentation — segment base = starting address in physical memory
- **Thrashing** — caused by excessive paging activity
- Buffer registers — overcome difference in data transfer speed

#### B. I/O Management
- Program-controlled I/O = repeatedly checking status flags (polling)
- Memory-mapped I/O = I/O and memory share same address space
- I/O-mapped I/O = separate address space

#### C. Disk Scheduling
- **FCFS (First Come First Serve)** — service requests in arrival order
- SSTF — service nearest request first
- SCAN — elevator algorithm
- Practice FCFS calculations:
  - Add up absolute differences between consecutive head positions

#### D. File Systems
- Types: NTFS, FAT32, HFS+ (all valid)
- Common file operations: Create, Delete, Rename (all)
- Secondary memory = long-term storage = **Logical** organization

#### E. Process Management
- Types of interrupts: Hardware, Software, Trap (NOT "Memory interrupt")
- Thread shares with same-process threads: **code section and data section**
- Types of OS: Real-time, Embedded, Network, Batch, Time-sharing

You must reach Level 4 for all TIER 1 topics before the exam.

## 10. MISTAKES TO AVOID

These are the most common mistakes that cost candidates marks:

### 10.1 Conceptual Traps (from PYQ analysis)

| Wrong Belief | Correct Fact |
|--------------|-------------|
| Peer-to-peer is a network topology | It is a network type, NOT topology |
| Fragmentation is a memory management technique | It is a PROBLEM, not a technique |
| Single-user access is a DBMS feature | Single-user access is actually a LIMITATION |
| Computer union is a SQL constraint | UNION is NOT a constraint; it's a set operation |
| COMPUTE is an aggregate function | COMPUTE is NOT valid; COUNT/SUM/AVG are valid |
| Private key in asymmetric cryptography = sender | Private key is kept by RECEIVER |
| Full access rights for all users = good security | This BREACHES security |
| Inline functions are large and complex | They are SMALL and simple; substituted at call point |
| When deleting from queue, check overflow | Check UNDERFLOW (is queue empty?) before deletion |
| Randomized traversal is a binary tree technique | It is NOT; Preorder/Inorder/Postorder are |
| IPv6 priority field is discarded in header translation | IPv6 FLOW LABEL is considered (not discarded) |
| delete ptr where ptr=NULL causes crash | It compiles and executes SUCCESSFULLY |

### 10.2 Exam Strategy Mistakes

- **Never skip Option (D) "More than one"** — approximately 20–25% of answers are (D)
- **Never skip Option (E) "None of the above"** — some correct answers are indeed (E)
- **Don't spend >90 seconds on any one question** — move on and return
- **Don't leave negative marking questions blank if TRE 4.0 has -¼** — attempt carefully with 70%+ confidence
- **Don't ignore Language section** — it is qualifying; failing it disqualifies you regardless of CS marks

## APPENDIX: QUICK FORMULA SHEET

### CS Formulas
```
Page offset bits = log₂(page size in bytes)
  4KB = 2¹² → 12 bits

MUX select lines = log₂(inputs)
  8:1 → log₂(8) = 3

Truth table rows = 2ⁿ (n = number of inputs)
  4 inputs → 2⁴ = 16

Catalan number C(n) = (2n)! / ((n+1)! × n!)
  C(4) = 14 (distinct BSTs from 4 keys)

FCFS Disk Scheduling:
  Total movement = Σ|position(i) - position(i-1)|

P = I²R (electrical power)

Spanning tree edges = N - 1 (for N vertices)

Binary search time = O(log N)
Merge sort time = O(N log N)
Bubble sort time = O(N²)
```

### Number Conversion Quick Reference
```
Decimal 1234:
  Binary  = 10011010010
  Octal   = 2322
  Hex     = 4D2

Binary to Decimal:
  1011.1101 = 8+2+1 + 0.5+0.25+0+0.0625 = 11.8125
```
